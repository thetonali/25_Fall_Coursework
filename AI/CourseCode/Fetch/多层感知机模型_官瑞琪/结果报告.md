## 3 数据预处理与特征提取分析

### 3.1 标签编码与格式标准化

将文本标签（'ham'/'spam'）转换为数值标签（0/1）。

![image-20260117153311962](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117153311962.png)

如图所示，处理结果为Ham (正常邮件) 映射为 0，Spam (垃圾邮件) 映射为 1。

这是监督学习的标准步骤。对于垃圾邮件过滤任务，通常将关注的少数类（垃圾邮件）设为正类（1），这便于后续计算 Precision和 Recall。

### 3.2 特征提取策略TF-IDF与N-gram

代码采用了TF-IDF结合N-gram的策略将非结构化文本转化为向量空间模型。

1. 选择TF-IDF的原因:
   1. TF(词频)反映词语在文档中的重要性。
   2. IDF(逆文档频率)降低常见词的权重,提升区分性词汇的权重。
   3. 相比简单的词袋模型,TF-IDF能更好地捕捉文本的语义特征。

2. 关键参数配置如下——

```bash
max_features=3000
```

​	将特征维度限制在3000维。这在保留核心语义信息的同时，有效防止了MLP模型因输入维度过高而产生“维	度灾难”或过拟合，同时也加快了训练速度。

```bash
ngram_range=(1,2)
```

 	使用1-gram和2-gram。不仅提取单个单词，还提取相邻的两个词。

​	例如"free entry"或 "call now"这样的词组比单独的 "free" 或 "call" 具有更强的垃圾邮件指示性。

```bash
stop_words='english'
```

​	移除英文停用词,去除了 'the', 'is' 等无实际意义的高频词，降低了噪声干扰。

![image-20260117153910444](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117153910444.png)|

如图所示，最终得到的特征矩阵为 `(5572, 3000)` 的稀疏矩阵,每个样本用3000维的TF-IDF向量表示。

## 4 MLP 模型构建与训练过程

### 4.1 MLP 模型构建与训练过程

构建了一个多层感知机（MLP）神经网络，其配置充分考虑了文本分类任务的复杂性。

1. 采用了(100, 50)双隐藏层架构。

   第一层100个神经元用于捕捉 TF-IDF 特征中的基础语义组合；
   第二层 50 个神经元进一步进行非线性特征压缩。

2. 使用 Adam 优化器，结合 ReLU 激活函数。

   Adam 能自适应调整学习率，确保训练初期快速收敛；
   ReLU 则有效缓解了深度网络中的梯度消失问题。

3. alpha=0.0001（L2 正则化）对权重进行惩罚，防止模型过度依赖某些特定噪声词汇。

4. 模型会自动从训练集中预留 10% 作为验证集，通过监控验证集性能来决定何时停止训练。

### 4.2 训练动态与收敛性

![image-20260117154653139](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117154653139.png)

如图所示，从终端输出的迭代过程可以看到——

1. 在第 1 次迭代时，Loss 为 0.3197；
   到第 3 次迭代时，Loss 迅速下降至 0.0191。

   这说明 TF-IDF 提取的特征具有极强的区分度，模型能迅速捕捉到判别模式。

2. 模型在第 16 次迭代时触发了早停机制。
   尽管 max_iter 设为 300，但实际仅用 16 次迭代即完成训练。

   这表明模型已经达到了性能瓶颈，继续训练可能会导致过拟合。

3. 最终训练损失为 0.002521。

   这是一个非常低的值，说明模型在训练集上几乎实现了完美拟合。同时，验证集分数（Validation Score）稳定在 0.982 左右，显示出极高的泛化潜力。

### 4.3 损失曲线

![image-20260117154930533](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117154930533.png)

如图所示，损失曲线呈现出典型的“L”型下降趋势。初期曲线斜率极大，随后迅速变平缓。

## 5 模型性能评估

### 5.1 核心指标

![image-20260117155307553](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117155307553.png)

如图所示，通过对比训练集和测试集的表现，模型展现了很高的分类水准。

1. 测试集准确率达到 98.03%。

   在如此不平衡的数据集中，这是一个很高的分数，说明模型对绝大多数邮件都能做出正确判断。

2. 垃圾邮件（Spam）的 F1 分数为 0.9247。

   由于 F1 是精确率和召回率的调和平均，该指标的高分证明模型在“不抓错”和“不漏抓”之间取得了很好的平衡。

### 5.2 混淆矩阵分析

![image-20260117155506788](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117155506788.png)

如图所示，

1. 真负例 (TN=958)，说明绝大多数正常邮件被正确放行。
2. 假正例 (FP=8)，说明仅有 8 条正常邮件被误判为垃圾邮件（误报率约为 0.8%）。在邮件过滤系统中，8/966 的超低误报率说明该 MLP 模型具有很高的安全性。
3. 假负例 (FN=14)，说明有 14 条垃圾邮件未能拦截。这属于“漏报”，虽然会给用户带来轻微骚扰，但相比误报而言，其代价较低。

### 5.3 拟合程度分析

![image-20260117155720167](C:\Users\Slavo\AppData\Roaming\Typora\typora-user-images\image-20260117155720167.png)

如图所示，

1. 训练集各项指标达到99%以上，接近完美拟合。

2. 测试集各项指标保持在 90% - 98% 之间。

   虽然训练集表现略高于测试集，这是深度学习的正常现象。但两者差距微小，这说明设置的 L2 正则化 (alpha=0.0001)和早停机制 (Early Stopping) 发挥了显著作用，成功抑制了过拟合，使模型具备了真正的预测能力。

3. 分类报告显示，垃圾邮件的 Recall (召回率) 为 90.60%，略低于 Precision (精确率) 94.41%。

   这说明模型在识别垃圾邮件时显得相对“谨慎”。这可能是因为某些垃圾邮件的用词非常接近正常沟通，或者是因为 TF-IDF 的 3000 维特征仍未能覆盖某些极其生僻的干扰词。

## 6 总结

本次作业构建了一个基于多层感知机（MLP）的垃圾邮件过滤系统。

通过 TF-IDF 特征提取和双层神经网络建模，模型在测试集上实现了 98.03% 的准确率和 94.41% 的垃圾邮件精确率。

说明神经网络在处理具有明显词汇特征（如数字、促销短语、特定 N-gram 组合）的文本分类任务时具有显著优势。

1. TF-IDF结合N-gram的策略能有效捕捉垃圾邮件中的高区分度信号（如数字、金额、促销短语），远优于简单的词频统计，说明好的特征工程对模型性能至关重要。

2. 感受到深度学习（MLP）在文本分类任务中的强大收敛能力。仅16次迭代即可达到极低损失，验证了神经网络对复杂模式的高效学习能力，但也要警惕过拟合风险。
