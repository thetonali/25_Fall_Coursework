## 3 算法参数选择

### 3.1 k-最近邻 (k-NN)

算法原理：

- 对于测试样本，找到训练集中距离最近的k个样本
- 根据这k个邻居的类别投票决定预测结果

基线参数（默认）：

```python
self.baseline_models['k-NN'] = {
    'model': KNeighborsClassifier(),  # n_neighbors=5
    'use_scaled': True
}
```

优化参数：

```python
self.optimized_models['k-NN'] = {
    'model': KNeighborsClassifier(
        n_neighbors=18,      # 增加邻居数以提高稳定性
        weights='distance',  # 距离加权，近邻影响更大
        metric='manhattan',  # 曼哈顿距离对异常值更鲁棒
        p=2
    ),
    'use_scaled': True
}
```

优化理由：

1. `n_neighbors=18`：贷款数据集样本量大(50,000)，增加邻居数可提高稳定性
2. `weights='distance'`：距离加权使近邻的投票权重更高
3. `metric='manhattan'`：曼哈顿距离对离群值不敏感

------

### 3.2 逻辑回归 (Logistic Regression)

算法原理：

使用sigmoid函数将线性组合映射到概率：

$$P(y=1|x) = \frac{1}{1 + e^{-(\beta_0 + \beta^T x)}}$$

通过最大化似然函数学习参数。

基线参数（默认）：

```python
self.baseline_models['Logistic Regression'] = {
    'model': LogisticRegression(max_iter=1000, random_state=42),
    'use_scaled': True
}
```

优化参数：

```python
self.optimized_models['Logistic Regression'] = {
    'model': LogisticRegression(
        C=3.7554011,      # 适度正则化
        penalty='l2',     # L2正则化
        solver='lbfgs',   # 适合大数据集
        max_iter=1000,
        random_state=42
    ),
    'use_scaled': True
}
```

优化理由：

1. `C=3.76`：适度增强正则化（C>1），在大数据集上平衡拟合和泛化。
2. `solver='lbfgs'`：对大规模数据效率高。

------

### 3.3 随机森林 (Random Forest)

算法原理：

集成学习方法，训练多棵决策树：

1. Bootstrap采样：从训练集有放回抽样。
2. 随机特征选择：每次分裂只考虑随机特征子集。
3. 投票决策：所有树的预测结果投票。

基线参数（默认）：

```python
self.baseline_models['Random Forest'] = {
    'model': RandomForestClassifier(random_state=42),
    'use_scaled': False
}
```

优化参数：

```python
self.optimized_models['Random Forest'] = {
    'model': RandomForestClassifier(
        n_estimators=196,       # 增加树数量提高稳定性
        max_depth=None,         # 不限制深度（数据量大）
        min_samples_split=5,    # 增加分裂阈值防止过拟合
        min_samples_leaf=4,     # 叶节点最小样本数
        max_features=None,      # 考虑所有特征
        random_state=42
    ),
    'use_scaled': False
}
```

优化理由：

1. `n_estimators=196`：增加树数量提高集成效果
2. `min_samples_split=5`：防止过拟合
3. `max_features=None`：数据量大，可考虑所有特征

------

### 3.4 多层感知机 (MLP)

算法原理：

多层神经网络，通过反向传播学习：

$$h = f(Wx + b)$$

其中 $f$ 是激活函数，本作业使用tanh。

基线参数（默认）：

```python
self.baseline_models['MLP'] = {
    'model': MLPClassifier(max_iter=1000, random_state=42),
    'use_scaled': True
}
```

优化参数：

```python
self.optimized_models['MLP'] = {
    'model': MLPClassifier(
        hidden_layer_sizes=(50,),    # 单层50个神经元
        activation='tanh',           # tanh激活函数
        alpha=0.008181203,          # 正则化参数
        learning_rate='constant',    # 恒定学习率
        random_state=42
    ),
    'use_scaled': True
}
```

优化理由：

1. `hidden_layer_sizes=(50,)`：简化网络防止过拟合
2. `activation='tanh'`：tanh在某些数据上收敛更好
3. `alpha=0.0082`：适度正则化

## 4 评估指标介绍

### 4.1 混淆矩阵

|                 | 预测Rejected(0) | 预测Approved(1) |
| --------------- | --------------- | --------------- |
| 实际Rejected(0) | TN (真负例)     | FP (假正例)     |
| 实际Approved(1) | FN (假负例)     | TP (真正例)     |

### 4.2 核心指标

#### 4.2.1 准确率 (Accuracy)

$$\text{Accuracy} = \frac{TP + TN}{TP + TN + FP + FN}$$

所有预测中正确的比例。目标：> 0.85

#### 4.2.2 精准率 (Precision)

$$\text{Precision} = \frac{TP}{TP + FP}$$

预测为Approved的样本中，真正是Approved的比例。衡量误批率。

#### 4.2.3 召回率 (Recall)

$$\text{Recall} = \frac{TP}{TP + FN}$$

实际是Approved的样本中，被正确预测的比例。衡量漏批率。

#### 4.2.4 F1分数

$$\text{F1} = 2 \cdot \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$

精准率和召回率的调和平均，综合评估模型性能。

### 4.3 交叉验证

使用5折交叉验证评估模型稳定性：

```python
cv_scores = cross_val_score(model, X_train, self.y_train, cv=5)
```

- 更全面地评估模型性能
- 检测过拟合问题
- 评估模型对数据划分的敏感度
