## 1 优化目标与策略

### 1.1 优化目标

1. 尽可能提高各个模型的准确率。
2. 减少过拟合，提高交叉验证分数从而提升泛化能力。
3. 同时提高精准率和召回率。

### 1.2 优化策略

1. 超参数调优
   1. 搜索方式——网格搜索/随机搜索
   2. 手动实验
2. 模型正则化
   1. 深度/复杂度限制
   2. L1/L2正则化
   3. 早停策略

#### **2.2.1 k-NN优化**

搜索得到的最佳参数为`n_neighbors:1，p:1，weight:uniform`。
除了准确率，其他三项指标也都得到了提高。

1. 减小邻居数可以使模型更敏感地捕捉到局部模式。

   但考虑到这个sonar数据集中样本数太少，减小邻居数容易受噪声点干扰。

   于是手动尝试了1,3两个较小值和7,9两个较大值(大于1时weight都采用距离加权)，最终发现较小值的表现更好。

2. metric使用余弦距离，在处理频率特征时可能更有效。

- 得到优化参数:

```python
self.optimized_models['k-NN'] = {
    'model': KNeighborsClassifier(n_neighbors=1, weights='uniform', 
                        		 metric='cosine', p=1),
     'use_scaled': True
}
```

#### **2.2.2 朴素贝叶斯优化**

两种搜索方式均未找到有效提升的参数，故选择分析基线模型的问题。

基线模型`GaussianNB()`为默认参数。

![image-20260108144737834](E:\Courses\25Fall\AI\大作业\基础作业_官瑞琪\assets\image-20260108144737834.png)

1. 朴素贝叶斯假设特征独立，但声呐数据集相邻频率的能量值高度相关(如图所示)。

   通过PCA转换出的主成分是正交的。

2. 基线模型无法捕捉特征之间的相互作用。

- 优化策略:

  1. 使用PCA降维去相关。

  2. 增大平滑因子。

     平滑公式： $\sigma^2_{smoothed} = \sigma^2 + \alpha \cdot \max(\sigma^2)$

     防止方差过小导致的数值下溢，但注意过大的平滑会损害模型精度。

- 得到优化参数：

  注：其中`n_components=0.65`与`var_smoothing=1e-7`手动测试了一部分范围后选中。

  ```python
  self.optimized_models['Naive Bayes'] = {
      'model': Pipeline([
      ('pca', PCA(n_components=0.65)),
      ('gnb', GaussianNB(var_smoothing=1e-7)) # 增大平滑因子
  ]),
  'use_scaled': True
  }
  ```

#### **2.2.4 随机森林优化**

搜索得到的最佳参数为：`max_depth:None，max_features:sqrt，min_sample_leaf:1，min_samples_split:5，n_estimators:50`。

- 优化策略：

  1. max_features='sqrt'：

     声纳数据集有 60 个特征，设置为 sqrt（约 8 个特征）强迫每棵树在分裂时只能从随机的 8 个特征中挑选。这增加了树与树之间的差异性，降低了模型间的相关性，从而更有效地通过集体投票抵消了个别树的误差。

  2. min_samples_split=5：

     基线模型默认是 2。在只有 200 个样本的情况下，如果两个样本靠得很近但类别不同（可能是测量噪声），默认参数会为此专门分出一个枝杈。改为 5 之后，模型被迫寻找更有代表性的规律，增强了泛化能力。

  3. n_estimators=50：

     对于极小数据集，树的数量不是越多越好。50 棵树已经足够通过“大数定律”产生稳定的投票结果，同时避免了过多的弱树引入不必要的随机干扰。

- 得到优化参数：

  ```python
  self.optimized_models['Random Forest'] = {
      'model': RandomForestClassifier(n_estimators=50, max_depth=None,
                                      max_features='sqrt', min_samples_leaf=1,
                                      min_samples_split=5, random_state=42),
      'use_scaled': False
  }
  ```

  直接使用网格搜索得到的最佳参数。

#### **2.2.5 逻辑回归优化**

根据搜索方式得到的最佳参数为：

`c:0.63386…,penalty:l2,solver:liblinear`。

1. C=0.638653441536259，为正则化强度的倒数。

   这里C<1(默认系数)，表示比默认更强的正则化。通过惩罚权重系数的大小，减少模型对某些特征的过度依赖。

   在该数据集小样本场景下，逻辑回归容易过拟合。增加正则化强度通常能提升泛化性能。

2. 使用L2正则化

   权重平方和惩罚。倾向于让权重变小但不为零。

3. solver='liblinear'

   对于小数据集和二分类问题，是最稳定高效的求解器。

- 得到优化参数：

  ```python
  self.optimized_models['Logistic Regression'] = {
      'model': LogisticRegression(C=0.64, penalty='l2', solver='liblinear',
                                  max_iter=1000, random_state=42),
      'use_scaled': True
  ```
