   [Quidway-Ethernet 0/0]rip version 2    % 配置rip2协议

   [Quidway-rip-1]network network-address   % 按照自然网段书写即可
2、学会使用基本访问控制列表：
   实验1：AR28-11路由器的访问控制列表或者说防火墙的缺省过滤方式是允许通过还是禁止通过？
   实验2：禁止特定主机PC3访问PC5所在网络内的主机。
   实验3：禁止特定网络         (即某pc所在的网络)内的主机访问PC5所在网络内的主机。
   
   创建基本的访问控制列表可由下面三个步骤完成，涉及的命令如下：
   (1) 在Router上配置安全区域和安全域间。
  <Huawei> system-view
  [Huawei] firewall zone trust           %配置可信域  
  [Huawei-zone-trust] priority 14
  [Huawei-zone-trust] quit
  [Huawei] firewall zone untrust       %配置不可信域      
  [Huawei-zone-untrust] priority 1
  [Huawei-zone-untrust] quit
  [Huawei] firewall interzone trust untrust   %配置安全域间
  [Huawei-interzone-trust-untrust] firewall enable  %使能防火墙功能。
  [Huawei-interzone-trust-untrust] packet-filter default  permit  inbound 

  [Huawei-interzone-trust-untrust]packet-filter default  permit  outbound 
  [Huawei-interzone-trust-untrust] quit
   
  执行命令display firewall zone [ zone-name ] [ interface | priority ] %查看安全域的相关信息。
  执行命令display firewall interzone [ zone-name1 zone-name2 ] %查看安全域间的相关信息。
  执行命令display firewall-nat session aging-time %查看防火墙会话表老化时间的相关信息。
(2)在Router上将接口加入安全区域。
[Huawei] interface gigabitethernet 3/0/0
[Huawei-GigabitEthernet3/0/0] ip address 10.39.2.1 24 
[Huawei-GigabitEthernet3/0/0] zone untrust
[Huawei-GigabitEthernet3/0/0] quit
(3)在Router上配置ACL
基础访问

[Huawei] acl 2102
[Huawei-acl-adv-2102] rule permit或deny source 10.39.2.3 0.0.0.0 
[Huawei-acl-adv-2102] rule permit或deny source 10.39.2.3 0.0.0.0 
[Huawei-acl-adv-2102] rule permit或deny source 10.39.2.3 0.0.0.0 


高级访问
[Huawei] acl 3102
[Huawei-acl-adv-3102] rule permit或deny tcp source 10.39.2.3 0.0.0.0 destination 10.38.1.2 0.0.0.0
[Huawei-acl-adv-3102] rule permit或deny tcp source 10.39.2.3 0.0.0.0 destination 10.38.1.3 0.0.0.0
[Huawei-acl-adv-3102] rule permit或deny tcp source 10.39.2.3 0.0.0.0 destination 10.38.1.4 0.0.0.0

[Huawei-acl-adv-3102] quit
(4)在Router上配置ACL
[Huawei] firewall interzone trust untrust
[Huawei-interzone-trust-untrust] packet-filter 3102 inbound
[Huawei-interzone-trust-untrust] quit
（5）验证配置结果。
配置成功后，仅特定主机（10.39.2.3）可以访问内部服务器。

在Router上执行display firewall interzone [ zone-name1 zone-name2 ]命令，结果如下。

3、学会使用高级访问控制列表：
   实验1：FTP访问控制实验；
     (1)、在PC5上搭建FTP server；
     (2)、测试各个主机是否能打开ftp；
     (3)、禁止特定主机cilent访问FTP；
     (4)、禁止特定网络201.1.1.0/192的所有主机访问FTP；
     (5)、改变FTP的端口为2121后，禁止特定网络201.1.1.0/192的所有主机访问FTP。
   涉及的命令如下：
   [Quidway] acl number 3000   %创建一个编号从3000~3999的高级访问控制列表
   [Q-a-a-3000] rule deny tcp source 201.1.1.4 0 destination 201.1.1.67 0 destinaton-port eq 21
   [Quidway-Ethernet 0/0] firewall packet-filter 3000 outbound
   [Quidway-Ethernet 0/0] undo firewall packet-filter 3000 outbound

   对于禁止特定网络内所有主机访问的ACL：
   [Q-a-a-3001] rule deny tcp source 201.1.1.0 0.0.0.63 destination 201.1.1.67 0 destination-port eq 21

   实验2：禁止使用QQ实验(假设PC5为QQ服务器，禁止网络上的主机访问QQ，即访问到特定主机PC5)。
     (1)、禁止特定主机PC3访问PC5：
     (2)、禁止特定网络201.1.1.0/192中的主机访问PC5
   涉及的命令如下：
   [Quidway] acl number 3000
   [Q-a-a-3000] rule deny icmp source 201.1.1.4 0 destination 201.1.1.67 0
   [Quidway-Ethernet 0/0] firewall packet-filter 3000 outbound
   [Quidway-Ethernet 0/0] undo firewall packet-filter 3000 outbound

   对于禁止特定网络内所有主机访问的ACL：
   [Q-a-a-3004] rule deny icmp source 201.1.1.0 0.0.0.63 destination 201.1.1.67 0

注意：1、接口标号与物理拓扑图上的对应关系： 
         Ethernet 0/0 <---> e0     Ethernet 0/1 <---> e1    Ethernet 1/0 <---> e2                 
      2、不需要配置s0、s1等串口的ip地址，以及相应网络上的数据链路层协议。  
      3、其他命令参照vrp文件夹下的“安全操作.pdf”中的访问控制列表配置说明。
