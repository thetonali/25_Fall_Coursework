# 计算机网络综合题设计

## 题目类型：分析与计算综合题

## 题目内容（总分：18分）

某企业需要为其5个主要部门（研发部、市场部、行政部、服务器区、DMZ区）建立内部网络，并对外连接到 Internet。企业获得的公网IP地址块是 $172.16.0.0/16$
请根据以下需求，完成子网规划、地址分配与路由计算。

**网络拓扑描述：**

- 企业总部网络地址：`172.16.0.0/16`
- 需要划分为5个子网：
  - 子网A（研发部门）：需要容纳500台主机
  - 子网B（市场部门）：需要容纳200台主机
  - 子网C（行政部门）：需要容纳100台主机
  - 子网D（服务器区）：需要容纳50台主机
  - 子网E（DMZ区）：需要容纳30台主机
- 路由器配置：企业内部有3台路由器（R1、R2、R3）组成核心网络
- R1连接子网A和子网B
- R2连接子网C和子网D
- R3连接子网E并作为边界路由器连接外网
- R1、R2、R3之间通过点对点链路互联

**路由信息：** 下表为R1路由器当前的路由表（简化版）：

| 目的网络   | 子网掩码        | 下一跳       | 接口 | 跃点数 |
| ---------- | --------------- | ------------ | ---- | ------ |
| 172.16.0.0 | 255.255.254.0   | 直连         | E0   | 0      |
| 172.16.2.0 | 255.255.255.0   | 直连         | E1   | 0      |
| 172.16.4.0 | 255.255.255.128 | 172.16.100.2 | S0   | 1      |
| 172.16.8.0 | 255.255.255.192 | 172.16.100.6 | S1   | 2      |

**问题：**

### （1）子网划分（6分）

请使用VLSM（可变长子网掩码）技术，为上述5个子网合理分配IP地址段。要求：

- 写出每个子网的网络地址、子网掩码（用CIDR表示）、可用IP地址范围、广播地址。
- 说明你的划分依据和顺序。

### （2）路由聚合（4分）

假设R2需要向R3通告子网A、B、C、D的路由信息。请问：

- R2能否对这些路由进行聚合？如果可以，写出聚合后的路由条目（网络地址/前缀长度）。
- 计算该聚合路由的子网掩码，并说明聚合的优缺点。

### （3）路由分析（5分）

根据R1的路由表回答：

- 如果子网A中的主机`172.16.0.10`要访问IP地址`172.16.4.50`，数据包在R1的转发过程是怎样的？请详细说明路由匹配过程。
- 路由表中的第四条路由`172.16.8.0/26`通过S1接口到达，跃点数为2，请解释这个跃点数的含义，并推测该路由可能经过了哪些路由器。

### （4）路由协议选择（3分）

该企业网络规模中等，需要选择合适的动态路由协议。请比较RIP、OSPF、BGP三种协议，说明：

- 哪种协议最适合该企业内部网络？（给出2个理由）。
- 如果R3需要与外部ISP路由器交换路由信息，应该使用什么协议？为什么？

## 详细答案及评分标准

### （1）子网划分（6分）

**解答步骤：**

VLSM划分原则：按主机数量从大到小依次分配，避免地址浪费。

**具体划分：**

1. **子网A（研发部门）- 需要500台主机**
   - 需要主机位数：2^9 = 512 > 500，需要9位主机位
   - 子网掩码：32-9 = 23位，即 `/23` 或 `255.255.254.0`
   - 网络地址：`172.16.0.0/23`
   - 可用IP范围：`172.16.0.1 - 172.16.1.254`
   - 广播地址：`172.16.1.255`
2. **子网B（市场部门）- 需要200台主机**
   - 需要主机位数：2^8 = 256 > 200，需要8位主机位
   - 子网掩码：`/24` 或 `255.255.255.0`
   - 网络地址：`172.16.2.0/24`
   - 可用IP范围：`172.16.2.1 - 172.16.2.254`
   - 广播地址：`172.16.2.255`
3. **子网C（行政部门）- 需要100台主机**
   - 需要主机位数：2^7 = 128 > 100，需要7位主机位
   - 子网掩码：`/25` 或 `255.255.255.128`
   - 网络地址：`172.16.3.0/25`
   - 可用IP范围：`172.16.3.1 - 172.16.3.126`
   - 广播地址：`172.16.3.127`
4. **子网D（服务器区）- 需要50台主机**
   - 需要主机位数：2^6 = 64 > 50，需要6位主机位
   - 子网掩码：`/26` 或 `255.255.255.192`
   - 网络地址：`172.16.4.0/26`
   - 可用IP范围：`172.16.4.1 - 172.16.4.62`
   - 广播地址：`172.16.4.63`
5. **子网E（DMZ区）- 需要30台主机**
   - 需要主机位数：2^5 = 32 > 30，需要5位主机位
   - 子网掩码：`/27` 或 `255.255.255.224`
   - 网络地址：`172.16.5.0/27`
   - 可用IP范围：`172.16.5.1 - 172.16.5.30`
   - 广播地址：`172.16.5.31`

**评分细则：**

- 正确说明VLSM划分原则（从大到小分配）：1分
- 每个子网的网络地址、子网掩码、可用IP范围、广播地址完全正确：各1分（5×1=5分）
- 如果子网掩码计算错误但方法正确，每个扣0.5分
- 如果IP范围有小错误（如忘记排除网络地址或广播地址），每个扣0.3分

### （2）路由聚合（4分）

**解答：**

**分析各子网的二进制表示：**

- 子网A: `172.16.0.0/23` = `10101100.00010000.0000000|0.00000000`
- 子网B: `172.16.2.0/24` = `10101100.00010000.0000001|0.00000000`
- 子网C: `172.16.3.0/25` = `10101100.00010000.00000011.0|0000000`
- 子网D: `172.16.4.0/26` = `10101100.00010000.00000100.00|000000`

**聚合分析：** 观察前缀，找到最长公共前缀：

- 前21位都相同：`10101100.00010000.00000`
- 但子网A从第22位开始是0，而子网D从第22位开始经过几位后变化

要覆盖`172.16.0.0`到`172.16.4.63`的所有地址：

- 需要覆盖的范围：`172.16.0.0` - `172.16.7.255`（包含所有子网）
- 最小聚合路由：`172.16.0.0/21`（前21位固定）

**聚合后的路由条目：** `172.16.0.0/21` 或 `172.16.0.0`，子网掩码 `255.255.248.0`

**优缺点：**

- 优点：
  - 减少路由表条目数量（4条路由聚合为1条）
  - 减少路由更新开销
  - 提高路由查找效率
- 缺点：
  - 包含了一些未使用的地址空间（如172.16.5.32-172.16.7.255）
  - 可能导致路由黑洞（如果这些未使用地址被错误路由）
  - 降低了路由精度

**评分细则：**

- 正确计算出聚合路由`172.16.0.0/21`：2分
- 正确写出子网掩码`255.255.248.0`：0.5分
- 说明聚合优点（至少2点）：0.75分
- 说明聚合缺点（至少1点）：0.75分
- 如果聚合结果错误但思路正确，给1分

### （3）路由分析（5分）

**解答：**

**第一问：路由匹配过程（3分）**

源地址：`172.16.0.10`，目的地址：`172.16.4.50`

**路由匹配步骤：**

1. 检查路由表第一条：`172.16.0.0/23`
   - 目的地址`172.16.4.50`与`172.16.0.0`，掩码`255.255.254.0` (/23)
   - 不匹配（172.16.4不在172.16.0.0-172.16.1.255范围内）
2. 检查路由表第二条：`172.16.2.0/24`
   - 目的地址`172.16.4.50`与`172.16.2.0/24`
   - 不匹配
3. 检查路由表第三条：`172.16.4.0/25` (255.255.255.128)
   - 将`172.16.4.50`与掩码`255.255.255.128`进行AND运算
   - `172.16.4.50` AND `255.255.255.128` = `172.16.4.0`
   - **匹配成功！**
4. **转发决策：**
   - 匹配到路由条目：`172.16.4.0/25`，下一跳：`172.16.100.2`，出接口：S0
   - R1将数据包从S0接口转发给下一跳路由器`172.16.100.2`（即R2）

**第二问：跃点数分析（2分）**

**跃点数含义：** 跃点数（Hop Count）为2表示从R1到达目的网络`172.16.8.0/26`需要经过2个路由器（或称2跳）。

**路由路径推测：**

- 数据包从R1出发
- 第1跳：R1 → 某中间路由器（可能是R2或R3）
- 第2跳：中间路由器 → 目的网络所在路由器
- 可能的路径：R1 → R3 → R2（如果172.16.8.0/26连接在R2后面）
- 或：R1 → R2 → R3（如果172.16.8.0/26连接在R3后面）

**评分细则：**

- 正确说明路由匹配步骤，逐条检查路由表：1分
- 正确识别匹配的路由条目`172.16.4.0/25`：1分
- 正确说明转发决策（下一跳和出接口）：1分
- 正确解释跃点数含义：1分
- 合理推测路由路径：1分
- 如果过程有逻辑错误但结论正确，酌情扣0.5-1分

### （4）路由协议选择（3分）

**解答：**

**第一问：企业内部路由协议选择（2分）**

**最适合的协议：OSPF（开放最短路径优先）**

**理由：**

1. **收敛速度快：** OSPF是链路状态协议，使用Dijkstra算法计算最短路径，网络拓扑变化时收敛速度快，适合企业网络对稳定性的要求
2. **无跳数限制：** RIP最大跳数为15跳，OSPF没有跳数限制，扩展性更好，适合中等规模网络的未来扩展
3. **支持VLSM和CIDR：** OSPF原生支持可变长子网掩码，与题目中的VLSM子网划分方案完美配合
4. **分层设计：** OSPF支持区域（Area）划分，可以将不同部门划分为不同区域，减少路由开销

**为什么不选RIP：**

- RIP是距离矢量协议，收敛慢
- 最大跳数限制为15
- 使用跳数作为度量，不考虑带宽

**为什么不选BGP：**

- BGP是外部网关协议，主要用于自治系统（AS）之间
- 配置复杂，资源消耗大
- 对于企业内部网络过于复杂

**第二问：外部路由协议（1分）**

**应使用：BGP（边界网关协议）**

**原因：**

1. **设计目的：** BGP专门设计用于自治系统之间的路由交换，R3作为边界路由器连接ISP，需要使用BGP
2. **策略路由：** BGP支持丰富的路由策略，可以根据企业需求控制流量路径
3. **可靠性：** BGP基于TCP（端口179），保证路由信息的可靠传输
4. **行业标准：** 互联网骨干网都使用BGP，与ISP对接必须使用BGP

**评分细则：**

- 正确选择OSPF作为内部路由协议：0.5分
- 给出2个合理的理由：1分（每个理由0.5分）
- 说明不选RIP或BGP的原因（可选，答出可加0.5分作为bonus）
- 正确选择BGP作为外部路由协议：0.5分
- 给出合理解释：0.5分
- 如果选择其他协议但理由充分合理，酌情给分

## 涉及的知识点总结

本题综合考查了以下网络层知识点：

1. **IP地址与子网划分**
   - VLSM（可变长子网掩码）技术
   - CIDR（无类域间路由）表示法
   - 子网掩码计算
   - 网络地址、广播地址、可用IP范围计算
2. **路由聚合（Route Aggregation）**
   - 路由汇总原理
   - 最长前缀匹配
   - 路由聚合的优缺点分析
3. **路由表与数据包转发**
   - 路由表结构与字段含义
   - 最长前缀匹配算法
   - 数据包转发过程
   - 跃点数（Hop Count）概念
4. **动态路由协议**
   - RIP（距离矢量协议）特点
   - OSPF（链路状态协议）特点
   - BGP（路径矢量协议）特点
   - IGP与EGP的区别与应用场景
5. **网络规划与设计**
   - 企业网络拓扑设计
   - 路由协议选择依据
   - 内部网络与外部网络互联
