# 计算机网络综合题设计

## 题目类型：分析与计算综合题

## 题目内容（总分：18分）

某企业网络拓扑如下所示，需要进行网络规划与路由配置：

**网络拓扑描述：**

- 企业总部网络地址：`172.16.0.0/16`
- 需要划分为5个子网：
  - 子网A（研发部门）：需要容纳500台主机
  - 子网B（市场部门）：需要容纳200台主机
  - 子网C（行政部门）：需要容纳100台主机
  - 子网D（服务器区）：需要容纳50台主机
  - 子网E（DMZ区）：需要容纳30台主机
- 路由器配置：企业内部有3台路由器（R1、R2、R3）组成核心网络
  - R1连接子网A和子网B
  - R2连接子网C和子网D
  - R3连接子网E并作为边界路由器连接外网
  - R1、R2、R3之间通过点对点链路互联

**路由信息：**

下表为R1路由器当前的路由表（简化版）：

| 目的网络   | 子网掩码        | 下一跳       | 接口 | 跃点数 |
| ---------- | --------------- | ------------ | ---- | ------ |
| 172.16.0.0 | 255.255.254.0   | 直连         | E0   | 0      |
| 172.16.2.0 | 255.255.255.0   | 直连         | E1   | 0      |
| 172.16.4.0 | 255.255.255.128 | 172.16.100.2 | S0   | 1      |
| 172.16.8.0 | 255.255.255.192 | 172.16.100.6 | S1   | 2      |

### （4）该企业网络规模中等，需要选择合适的动态路由协议。请比较RIP、OSPF、BGP三种协议，说明：（3分）

### 问题2答案（4分）

**分析各子网的二进制表示：**

- 子网A: `172.16.0.0/23` = `10101100.00010000.0000000|0.00000000`
- 子网B: `172.16.2.0/24` = `10101100.00010000.0000001|0.00000000`
- 子网C: `172.16.3.0/25` = `10101100.00010000.00000011.0|0000000`
- 子网D: `172.16.4.0/26` = `10101100.00010000.00000100.00|000000`

-----------------（0.5分）

**聚合分析：**

观察前缀，找到最长公共前缀：前21位相同为`10101100.00010000.00000`

要覆盖从`172.16.0.0`到`172.16.4.63`的所有地址，需要包含`172.16.0.0 ~ 172.16.7.255`的范围。

-----------------（0.5分）

**聚合结果：**

**聚合后的路由条目：** `172.16.0.0/21`

**子网掩码：** `255.255.248.0`

-----------------（1分）

**聚合的优缺点：**

**优点：**-----------------（1分）

- 减少路由表条目数量（4条路由聚合为1条），降低路由器内存开销
- 减少路由更新信息的传播开销，提高网络效率
- 提高路由查找速度，加快数据包转发

**缺点：**-----------------（1分）

- 包含了一些未使用的地址空间（如`172.16.5.32 ~ 172.16.7.255`）
- 可能导致路由黑洞（如果这些未使用地址被错误路由）
- 降低了路由精度，失去了细粒度的路由控制能力

**【评分说明】**

- 正确分析二进制表示或找到公共前缀：0.5分
- 正确说明聚合范围分析：0.5分
- 正确写出聚合路由`172.16.0.0/21`：0.5分
- 正确写出子网掩码`255.255.248.0`：0.5分
- 说明聚合优点（至少2点）：1分
- 说明聚合缺点（至少1点）：1分
- 若聚合结果错误但思路正确，给1分

### 问题3答案（5分）

#### 第一小问：路由匹配过程（3分）

**源地址：** `172.16.0.10`
 **目的地址：** `172.16.4.50`

**路由匹配步骤：**-----------------（2分）

**步骤1：** 检查路由表第一条：`172.16.0.0/23`（掩码`255.255.254.0`）

- 将目的地址`172.16.4.50`与掩码进行AND运算
- 结果不匹配（`172.16.4`不在`172.16.0.0 ~ 172.16.1.255`范围内）

**步骤2：** 检查路由表第二条：`172.16.2.0/24`

- 将目的地址`172.16.4.50`与掩码`255.255.255.0`进行AND运算
- 结果不匹配

**步骤3：** 检查路由表第三条：`172.16.4.0/25`（掩码`255.255.255.128`）

- `172.16.4.50` AND `255.255.255.128` = `172.16.4.0`
- **匹配成功！**

**转发决策：**-----------------（1分）

- 匹配到路由条目：`172.16.4.0/25`
- **下一跳：** `172.16.100.2`
- **出接口：** S0
- R1将数据包从S0接口转发给下一跳路由器`172.16.100.2`（即R2）

#### 第二小问：跃点数分析（2分）

**跃点数含义：**-----------------（1分）

跃点数（Hop Count）为2表示从R1到达目的网络`172.16.8.0/26`需要经过2个路由器（或称2跳）。即数据包需要经过2次转发才能到达目的网络。

**路由路径推测：**-----------------（1分）

数据包从R1出发，经过以下可能的路径：

- **第1跳：** R1 → 某中间路由器（可能是R2或R3）
- **第2跳：** 中间路由器 → 目的网络所在路由器

**可能的路径：**

- 路径1：R1 → R3 → R2（如果`172.16.8.0/26`连接在R2）
- 路径2：R1 → R2 → R3（如果`172.16.8.0/26`连接在R3）

**【评分说明】**

- 正确解释跃点数含义（经过的路由器数量）：1分
- 合理推测路由路径：1分
- 若只答出跃点数含义但未推测路径，得1分

### 问题4答案（3分）

#### 第一小问：企业内部路由协议选择（2分）

**最适合的协议：OSPF（开放最短路径优先）**-----------------（0.5分）

**理由：**-----------------（1.5分）

**理由1：收敛速度快**

- OSPF是链路状态协议，使用Dijkstra算法计算最短路径
- 网络拓扑变化时收敛速度快，适合企业网络对稳定性和快速恢复的要求

**理由2：无跳数限制，扩展性好**

- RIP最大跳数为15跳，限制了网络规模
- OSPF没有跳数限制，适合中等规模网络的未来扩展

**理由3：支持VLSM和CIDR**

- OSPF原生支持可变长子网掩码
- 与题目中的VLSM子网划分方案完美配合，能够准确传递子网信息

**理由4：支持分层设计**

- OSPF支持区域（Area）划分
- 可以将不同部门划分为不同区域，减少路由开销，提高网络可管理性

**【说明】** 答出其中任意2个理由即可得满分

**【评分说明】**

- 正确选择OSPF：0.5分
- 每个合理理由：0.75分（答出2个即得满分1.5分）
- 若选择其他协议但理由充分合理，酌情给分

#### 第二小问：外部路由协议（1分）

**应使用协议：BGP（边界网关协议）**-----------------（0.5分）

**原因：**-----------------（0.5分）

- **设计目的：** BGP专门设计用于自治系统（AS）之间的路由交换，R3作为边界路由器连接ISP，需要使用BGP
- **策略路由：** BGP支持丰富的路由策略和路径属性，可以根据企业需求控制流量路径和实施路由策略
- **可靠性：** BGP基于TCP协议（端口179），保证路由信息的可靠传输
- **行业标准：** 互联网骨干网都使用BGP，与ISP对接必须使用BGP协议

**【说明】** 答出其中任意1-2个原因即可

   - 数据包转发过程
   - 跃点数（Hop Count）概念

   - RIP（距离矢量协议）特点与限制
   - OSPF（链路状态协议）特点与优势
   - BGP（路径矢量协议）特点与应用场景

5. **网络规划与设计**
   - 企业网络拓扑设计
