### 第二章《数据科学过程》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **简述数据科学过程的六个主要步骤，并指出哪一个步骤通常占据最多的时间，为什么？**
    *   **考点关联：** PPT第2页、第4-83页的整体架构。
    *   **参考答案：**
        六个步骤为：
        1.  **设置研究目标**：定义项目做什么、为什么做及如何做（创立项目章程）。
        2.  **检索数据**：查找并获取所需的内部或第三方数据。
        3.  **数据准备**：包括数据清洗、整合、转换和规约，是确保数据质量的关键。
        4.  **数据探索（EDA）**：使用可视化和统计技术理解变量结构和关系。
        5.  **数据建模**：选择算法、执行模型、诊断与比较。
        6.  **展示与自动化**：结果可视化及应用落地。
        *通常“数据准备”占据最多时间（约60%-80%）。原因在于：现实世界的数据往往是“脏”的，存在缺失、噪声、不一致等问题（垃圾输入导致垃圾输出），且需要大量领域知识来转换特征以匹配模型需求。*

2.  **在数据清洗过程中，什么是“异常值”（Outliers）？请列举PPT中提到的两种检测异常值的方法。**
    *   **考点关联：** PPT第22-24页。
    *   **参考答案：**
        异常值是指某些观测值远离其他观测值，遵循不同的逻辑或生成过程的数据点。
        检测方法：
        1.  **可视化法**：使用标有最小值和最大值的图表，如**散点图**（观察偏离回归线的点）或**箱线图**（Boxplot，观察超出触须范围的点）。
        2.  **直方图分布**：观察分布两端极少出现的孤立数值。
        3.  **统计逻辑检查**：如根据业务逻辑判断“不可能的数值”（如年龄299岁）。

3.  **请解释“归一化”（Normalization）与“标准化”（Standardization）的区别，并说明在什么情况下标准化优于归一化？**
    *   **考点关联：** PPT第39-42页。
    *   **参考答案：**
        *   **归一化**：将数据缩放到固定范围（通常是[0, 1]），公式为 $(X - X_{min}) / (X_{max} - X_{min})$。
        *   **标准化**：将数据转换为均值为0，方差为1的分布，公式为 $z = (x - \mu) / \sigma$。
        *   **标准化的优势**：当数据中存在**异常值**时，归一化会将正常样本“挤”在一起（因为最大值被异常值撑大），导致样本间距丢失；而标准化对异常值不敏感，能更好地保持样本间的距离结构，且更符合很多统计模型对数据服从正态分布的假设。

4.  **在模型评估中，混淆矩阵（Confusion Matrix）的四个基础指标TP、TN、FP、FN分别代表什么？**
    *   **考点关联：** PPT第80页。
    *   **参考答案：**
        *   **TP (True Positive, 真正例)**：真实为正类，模型预测也为正类（如：有狼，预测有狼）。
        *   **TN (True Negative, 真负例)**：真实为负类，模型预测也为负类（如：没狼，预测没狼）。
        *   **FP (False Positive, 假正例)**：真实为负类，模型错误预测为正类（如：没狼，预测有狼，即误报）。
        *   **FN (False Negative, 假负例)**：真实为正类，模型错误预测为负类（如：有狼，预测没狼，即漏报）。

5.  **什么是假设检验中的P值（P-value）？当P < 0.05时意味着什么？**
    *   **考点关联：** PPT第65-70页。
    *   **参考答案：**
        P值是一个概率值，表示在**无效假设（H0）**成立的前提下，观察到当前样本（或更极端情况）出现的概率。
        *   当 P < 0.05 时，意味着这是一个**小概率事件**。在统计学上，我们有理由拒绝无效假设（H0），认为预测变量对结果有**显著影响**（Statistically Significant）。

#### 二、 计算题（共2题，每题10分，需使用科学计算器）

**1. 数据预处理计算：标准化与距离**
假设你有一个包含三个样本的数据集，其中变量 $X$（年龄）的数值为：$[20, 30, 40]$。
请计算：
1.  该变量 $X$ 的均值（$\mu$）和总体标准差（$\sigma$）。
2.  对数值 $30$ 进行**标准化（z-score）**处理后的结果。
3.  若有另一个样本 $Y=[18, 28, 38]$，请计算样本X与样本Y在二维空间（假设只有这一维特征对应相减，实际上是点对点）的**欧氏距离**（即计算向量 $X$ 和 $Y$ 之间的距离，PPT第43页公式）。

*   **详细解答：**
    1.  **均值** $\mu = (20+30+40)/3 = 30$
        **方差** $\sigma^2 = [ (20-30)^2 + (30-30)^2 + (40-30)^2 ] / 3 = (100 + 0 + 100) / 3 = 66.67$
        **标准差** $\sigma = \sqrt{66.67} \approx 8.165$
        *(注：如果是样本标准差分母为n-1=2，则 $\sigma=10$。PPT公式未明确区分，通常标准化使用总体标准差，但考试时最好写出假设或公式)*
    2.  **标准化** $Z = (x - \mu) / \sigma$
        对于 $x=30$：$Z = (30 - 30) / 8.165 = 0$。
    3.  **欧氏距离**
        $D = \sqrt{(20-18)^2 + (30-28)^2 + (40-38)^2}$
        $D = \sqrt{2^2 + 2^2 + 2^2} = \sqrt{4+4+4} = \sqrt{12} \approx 3.464$

**2. 模型评估计算：分类指标**
某疾病检测模型的混淆矩阵如下表所示（总样本数 N=1000）：

|                | 预测：患病 (Positive) | 预测：健康 (Negative) |
| :------------- | :-------------------: | :-------------------: |
| **真实：患病** |        80 (TP)        |        20 (FN)        |
| **真实：健康** |       100 (FP)        |       800 (TN)        |

请计算以下指标（保留小数点后三位）：
1.  **准确率 (Accuracy)**
2.  **精确率 (Precision/查准率)**
3.  **召回率 (Recall/查全率)**
4.  **F1-Measure**

*   **详细解答：** (参考PPT第81页公式)
    1.  **Accuracy** = $(TP + TN) / Total$
        $= (80 + 800) / 1000 = 880 / 1000 = 0.880$
    2.  **Precision** = $TP / (TP + FP)$
        $= 80 / (80 + 100) = 80 / 180 \approx 0.444$
    3.  **Recall** = $TP / (TP + FN)$
        $= 80 / (80 + 20) = 80 / 100 = 0.800$
    4.  **F1-Measure** = $(2 * Precision * Recall) / (Precision + Recall)$
        $= (2 * 0.444 * 0.800) / (0.444 + 0.800)$
        $= 0.7104 / 1.244 \approx 0.571$

#### 三、 综合分析题（共2题，每题15分）

**1. 数据清洗与集成场景分析**
**背景：** 你正在处理某电商平台的销售数据，旨在建立预测销量的模型。数据分布在两张表中：表A是“订单表”（含订单ID、用户ID、日期、商品名），表B是“用户表”（含用户ID、年龄、性别、注册地）。
**问题：**
1.  **整合（Integration）：** 你需要将这两张表合并，请问应该使用PPT中提到的哪种操作（连接/堆叠/聚合）？为什么？
2.  **清洗（Cleaning）：** 合并后发现“年龄”字段有以下情况：① 部分值为Null；② 某用户年龄为“-5”；③ 某用户年龄为“200”。请分别根据PPT内容给出处理建议。
3.  **转换（Transformation）：** “性别”字段为文本（"Male", "Female"），但在建模时计算机只能处理数字。请结合PPT提出一种转换方法。

*   **详细解答：**
    1.  **整合操作：** 应使用**连接（Join/Merge）**（PPT第31-32页）。
        *理由：* 我们需要基于共同的键（用户ID）将用户的属性（年龄、性别）补充到订单数据中，以丰富单一观察结果的信息量，而不是简单地增加行数（堆叠）。
    2.  **清洗策略：**
        *   **Null (缺失值)：** 可采用**均值/中位数填补**（PPT第26页“依据估计计算”），或者如果缺失比例极低且数据量大，可以直接**删除观测值**。
        *   **-5 (输入错误)：** 这属于“不可能的值”（PPT第21页），处理方法通常是**手动覆盖**或视为缺失值进行插补，或者直接删除。
        *   **200 (异常值)：** 违反了“合理性检查”（check 0 <= age <= 120），应作为**异常值**处理，验证是否为录入错误，若无法验证则删除或插补。
    3.  **转换方法：** 使用**虚拟变量（Dummy Variables）**（PPT第46页）。
        *方法：* 将“性别”列拆分为两列（或一列），如 `Is_Male` (1为是，0为否) 和 `Is_Female`。这解决了分类变量无法直接运算的问题。

**2. 模型选择与结果解释**
**背景：** 你使用线性回归模型预测房价，输入变量包括房屋面积(x1)和卧室数量(x2)。
**结果：** 训练后得到模型 $y = 0.8*x1 + 5*x2 + 10$。统计输出显示：$R^2 = 0.92$，x1的P值为 0.001，x2的P值为 0.35。
**问题：**
1.  请解释 $R^2 = 0.92$ 的商业含义（参考PPT第63页）。
2.  根据P值判断，哪个变量对房价的影响是“显著”的？哪个是不显著的？如果你需要优化模型（减少变量），你会保留哪一个？（参考PPT第65页）。
3.  如果引入过多的变量（例如100个特征），$R^2$ 通常会变高，但这为什么不好？PPT中提到了什么指标来惩罚这种情况？

*   **详细解答：**
    1.  **$R^2$ 含义：** 表示模型拟合度很好。$R^2=0.92$ 意味着模型能够解释目标变量（房价）92%的变化量。在商业模式中，通常0.85以上被认为是好的模型，0.9以上具有很强的竞争力。
    2.  **显著性判断与优化：**
        *   x1 (P=0.001 < 0.05)：**极显著**，说明面积对房价有实质影响。
        *   x2 (P=0.35 > 0.05)：**不显著**，没有足够证据表明卧室数量独立地影响房价（可能与面积高度相关）。
        *   *优化：* 应保留x1，剔除x2（或进行降维/特征选择），因为x2不显著且增加变量会增加模型复杂度。
    3.  **过多变量的问题：** 引入过多变量会导致模型过度复杂（过拟合），虽然原始 $R^2$ 会升高，但模型泛化能力变差。
    *指标：* 应使用**调整后的可决系数（Adjusted R-squared）**。它会对模型复杂度（变量数量）进行惩罚，如果新引入的变量没有显著提升模型解释力，调整后的 $R^2$ 反而会下降。

#### 四、 论述题（1题，20分）—— *押题重难点*

**题目：在PPT第14页提到“垃圾输入导致垃圾输出（GIGO）”，并在第4-46页花费了近一半的篇幅讲述数据的检索、准备、清洗和转换。请结合这一结构安排，论述以下观点：**
**“在数据科学项目中，为什么数据预处理（Pre-processing）比模型构建（Modeling）更决定项目的成败？请从数据质量对模型假设的影响（如线性关系、正态分布）以及模型的可解释性两个角度进行深入阐述。”**

*   **答题思路与详细要点（建议分段作答）：**

    **1. 引言：GIGO原则的核心地位**
    *   开篇点题：数据科学不是魔法，模型的本质是寻找数据中的规律。如果输入数据（Input）充满噪声、错误或偏差，无论算法（Model）多么高级（如深度学习），其输出（Output）必然是错误的或不可用的。PPT中强调“尽早发现并纠正错误”正是为了避免后续的连锁反应。

    **2. 角度一：数据质量对模型假设的致命影响**
    *   **线性假设的破坏：** PPT第37页展示了 $log(x)$ 转换。如果在预处理阶段没有发现数据是非线性的（如指数增长），直接强行使用线性回归模型，模型将完全失效（欠拟合）。预处理中的**数据变换**是让数据“适配”模型假设的前提。
    *   **分布假设的破坏：** 许多统计模型（如线性回归的残差分析）假设数据服从正态分布。PPT第39-42页强调了**标准化**的重要性。如果数据存在严重的**异常值**（第22页）或**量纲差异巨大**（第28页），基于距离的算法（如KNN，第40页）会完全被大数值特征主导，导致模型偏差。不做清洗和归一化，模型就无法正确“看”到数据的真实结构。

    **3. 角度二：维度灾难与模型的可解释性**
    *   **维度灾难：** PPT第43页提到“变量太多...模型难以被控制”。如果在预处理阶段不进行**数据规约**（如PCA或相关性分析剔除冗余变量），模型不仅计算量大，而且容易过拟合（学到了噪音而非规律）。
    *   **可解释性：** PPT第64页提到“检测影响比完美拟合更重要”。如果数据没有经过良好的清洗（例如混入了错误的标签或未处理的混杂变量），我们得到的系数（Coefficients）可能完全是虚假的关联。只有高质量、经过**合理性检查**（第21页）的数据，才能产出可信的业务洞察（Business Insight）。

    **4. 结论**
    *   总结：模型构建（Step 5）往往只是几行代码（`model.fit()`），但其有效性完全依赖于前序步骤（Step 2, 3, 4）。数据预处理是地基，模型是楼房。地基不稳，大厦将倾。因此，数据科学家80%的时间花在清洗和准备数据上是合理且必要的，这是确保项目从“理论可行”走向“落地应用”的关键。

### 第三章《机器学习》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **简述监督学习（Supervised Learning）、无监督学习（Unsupervised Learning）和半监督学习（Semi-supervised Learning）的区别，并各举一个典型的算法或应用场景。**
    *   **参考答案：**
        *   **监督学习**：输入数据带有标签（Label/Answer），模型通过学习输入与输出的映射关系来进行预测。
            *   *例子*：线性回归（预测房价）、SVM（图像分类）。
        *   **无监督学习**：输入数据没有标签，模型通过分析数据的内在结构、分布或规律来进行学习。
            *   *例子*：K-Means（客户分群）、PCA（降维去噪）。
        *   **半监督学习**：介于两者之间，使用少量有标签数据和大量无标签数据进行训练。通常利用聚类或标签传播技术。
            *   *例子*：标签传播算法（Label Spreading），用于标注成本高昂的场景（如医学图像分析）。

2.  **什么是“过拟合”（Overfitting）？在模型构建过程中，有哪些常用的方法可以防止过拟合？（至少列举三种）**
    *   **参考答案：**
        *   **定义**：过拟合是指模型在训练数据上表现非常好（误差极低），但在未知的测试数据上表现很差（泛化能力弱）。模型学到了数据中的噪声而非普遍规律。
        *   **防止方法**：
            1.  **正则化（Regularization）**：如L1（Lasso）或L2（Ridge）正则化，通过引入惩罚项限制模型复杂度，减少特征权重。
            2.  **交叉验证（Cross-Validation）**：如K-折交叉验证，充分利用数据评估模型稳定性。
            3.  **增加训练数据**：更多的数据能让模型学到更真实的分布。
            4.  **剪枝（Pruning）**：在决策树中限制树的深度或叶节点数量。

3.  **请比较集成学习（Ensemble Learning）中Bagging和Boosting两种策略的主要区别（从样本选择、样例权重、计算方式三个维度回答）。**
    *   **参考答案：**
        *   **样本选择**：Bagging采用有放回的随机抽样（Bootstraping），训练集之间相互独立；Boosting每一轮的训练集不变，但样本权重会变化。
        *   **样例权重**：Bagging中每个样例权重相等；Boosting会增加前一轮分错样本的权重，减少分对样本的权重（关注“难题”）。
        *   **计算方式**：Bagging的各个模型可以**并行**生成（独立训练）；Boosting的模型必须**顺序**生成（后一个模型依赖前一个的结果）。

4.  **在支持向量机（SVM）中，什么是“间隔最大化”？引入“核函数”（Kernel）的主要目的是什么？**
    *   **参考答案：**
        *   **间隔最大化**：SVM试图寻找一个超平面，使得正负样本中最靠近超平面的点（支持向量）到超平面的距离最大。间隔越大，模型的鲁棒性和泛化能力通常越强。
        *   **核函数目的**：解决线性不可分问题。通过核函数将低维空间的非线性数据映射到高维特征空间，使其在高维空间中变得线性可分。

5.  **简述K-均值（K-Means）聚类算法的一个主要缺点，以及它是如何通过“初始值”影响聚类结果的？**
    *   **参考答案：**
        *   **主要缺点**：需要预先指定聚类的数量 $k$，且对异常值和噪声敏感。
        *   **初始值影响**：K-Means对初始质心（Centroids）的位置非常敏感。如果初始点选得不好（例如两个初始点选在了同一个簇中），可能导致算法陷入局部最优解，得到错误的聚类结果。因此通常需要多次运行或指定种子（Seed）来固定结果。

#### 二、 计算题（共2题，每题10分，需科学计算器）

**1. 回归模型评估：MSE与MAPE计算**
某数据科学家预测了5套房屋的价格（单位：万），真实值（$Y$）与模型预测值（$\hat{Y}$）如下表所示：

| 样本编号 | 真实价格 $Y$ | 预测价格 $\hat{Y}$ |
| :------- | :----------- | :----------------- |
| 1        | 100          | 110                |
| 2        | 200          | 190                |
| 3        | 150          | 165                |
| 4        | 300          | 270                |
| 5        | 250          | 250                |

请计算该模型的以下误差指标（计算过程保留小数点后两位）：
1.  **均方误差 (MSE, Mean Squared Error)**
2.  **平均绝对百分比误差 (MAPE, Mean Absolute Percentage Error)**

*   **详细解答：**
    **步骤1：计算每个样本的误差**
    *   样本1: 误差 $e = 100-110 = -10$, 绝对百分比 $= |-10|/100 = 0.10$
    *   样本2: 误差 $e = 200-190 = 10$, 绝对百分比 $= |10|/200 = 0.05$
    *   样本3: 误差 $e = 150-165 = -15$, 绝对百分比 $= |-15|/150 = 0.10$

    $Sum\ of\ Squared\ Errors = (-10)^2 + (10)^2 + (-15)^2 + (30)^2 + (0)^2$
    $= 100 + 100 + 225 + 900 + 0 = 1325$
    $MSE = 1325 / 5 = \mathbf{265.00}$

    **2. 计算 MAPE**
    公式：$MAPE = \frac{100\%}{n}\sum_{i=1}^{n} |\frac{Y_i - \hat{Y}_i}{Y_i}|$
    $Sum\ of\ \%Errors = 0.10 + 0.05 + 0.10 + 0.10 + 0.00 = 0.35$
    $MAPE = (0.35 / 5) * 100\% = \mathbf{7.00\%}$

**2. KNN分类与距离计算**

*   点A $(2, 3)$，类别：正类 (+)

#### 三、 综合分析题（共2题，每题15分）

**1. 主成分分析（PCA）与降维分析**
**背景：** 你正在处理一个包含“语文成绩”和“数学成绩”的学生数据集（如PPT第83-86页所示）。你发现这两个变量之间存在很强的正相关性（语文好的数学通常也好）。你决定使用PCA进行降维。
**问题：**
1.  **原理理解：** PCA的核心思想是“将数据沿方差最大方向投影”。请用通俗的语言解释，为什么我们要选择“方差最大”的方向？这与保留信息量有什么关系？
2.  **结果解读：** 假设你对某包含10个变量的数据集进行PCA，得到的碎石图（Scree Plot）显示：第一主成分解释了50%的方差，第二主成分解释了30%的方差，第三主成分解释了5%，后续成分逐渐递减。
    *   如果你想把数据压缩到2维，你会保留多少原始信息？
    *   这种压缩相比于直接删除后8个变量（只保留前两个原始变量）有什么优势？
3.  **变量含义：** PCA生成的新变量（如 $Lv1$）通常是原始变量的线性组合（例如 $Lv1 = 0.5*X1 + 0.8*X2$）。这给模型的**可解释性**带来了什么挑战？

*   **详细解答：**
    1.  **方差与信息量：** 方差大意味着数据在该方向上分布得越分散，区分度越高。如果数据挤在一起（方差小），我们很难区分不同的样本。因此，选择方差最大的方向投影，就是为了在降维的同时，**尽可能保留数据之间的差异性（即信息量）**，让数据更容易被区分。
    2.  **结果解读：**
        *   **保留信息：** 第一主成分(50%) + 第二主成分(30%) = **80%**。
        *   **优势：** 直接删除变量会彻底丢失某些特征的信息。而PCA生成的两个主成分是所有原始10个变量的**线性组合（融合）**。这意味着保留的前两个主成分实际上包含了所有10个变量中最关键的变化模式，而不是简单地丢弃信息。
    3.  **可解释性挑战：** 原始变量（如“含糖量”、“酸度”）具有明确的物理或业务意义。PCA生成的新变量（如“主成分1”）是这些物理量的数学混合体，很难直接用业务语言定义它代表什么（例如，它可能既代表了一点酸度，又代表了一点含糖量），导致模型变成了“黑盒”，难以向业务人员解释。

**2. 不平衡数据集与模型评估**
**背景：** 你建立了一个预测“罕见传染病”的分类模型（有监督学习）。测试集共有1000人，其中只有10人是真患者（正例），990人是健康人（负例）。你的模型将所有人全部预测为“健康”。
**问题：**
1.  **准确率陷阱：** 请计算该模型的准确率（Accuracy）。这个高准确率说明模型很好吗？为什么？
2.  **指标选择：** 在这种医疗场景下，我们更应该关注“精确率（Precision）”还是“召回率（Recall）”？请结合混淆矩阵的概念（TP/FP/FN/TN）解释原因。
3.  **改进策略：** 如果想改善对罕见病人的识别能力，在模型训练或数据准备阶段可以采用什么策略？（提示：参考Boosting或重采样技术）。

*   **详细解答：**
    1.  **准确率计算：**
        *   预测健康（负例），实际健康（负例）= TN = 990。
        *   预测健康（负例），实际患病（正例）= FN = 10。
        *   Accuracy = $(TP+TN)/Total = (0+990)/1000 = \mathbf{99\%}$。
        *   **分析：** 这个模型**非常糟糕**。尽管准确率高达99%，但它没有抓出一个真正的病人（TP=0），在医疗诊断中是毫无价值的。这说明在类别极度不平衡时，准确率失效。
    2.  **指标选择：** 应该更关注 **召回率（Recall）**。
        *   Recall = $TP / (TP + FN)$。
        *   在传染病控制中，**漏报（FN，有病没查出来）** 的代价极其高昂（可能导致疫情扩散）。我们宁可接受一定的**误报（FP，没病被隔离）**，也不能漏掉病人。因此需要最大化Recall。
    3.  **改进策略：**
        *   **数据层面**：使用**过采样（Over-sampling）**增加正样本（患者）的数量，或**欠采样（Under-sampling）**减少负样本数量，使数据平衡。
        *   **算法层面**：使用 **Boosting** 算法（如AdaBoost），因为它会自动提升那些“难以分类”（即被分错的少数类）样本的权重，迫使模型去关注稀有的正例。

#### 四、 论述题（1题，20分）

**题目：在数据科学项目中，人们常说“数据和特征决定了机器学习的上限，而模型和算法只是逼近这个上限”。请结合本章学习的“特征工程”、“模型构建过程”以及“偏差-方差权衡（Bias-Variance Tradeoff，即欠拟合与过拟合）”的概念，论述这句话的含义。**

*   **答题思路与核心要点：**

    **1. 开篇：解释名言的含义（3分）**
    *   这句话强调了**特征工程（数据质量）**的重要性高于**算法选择**。无论算法多么复杂（如深度神经网络），如果输入的数据（垃圾输入）没有包含足够的信息或特征提取不当，模型永远无法做出准确预测（垃圾输出）。

    **2. 特征工程的关键作用（6分）**
    *   **信息提取**：机器学习算法本质上是在寻找输入特征与输出目标之间的数学关系。如果特征工程（如PPT中提到的转换、组合、虚拟变量、降维）做得好，输入数据能清晰表达业务逻辑，那么即使使用简单的算法（如线性回归）也能得到很好的结果。
    *   **易得性偏差**：PPT提到“最严重的错误是易得性偏差”，即只用容易拿到的数据建模。这会导致模型片面。好的特征工程需要挖掘“潜在变量”（如PCA发现的变量），这才是决定模型上限的关键。

    **3. 模型与算法的作用（逼近上限）（6分）**
    *   **欠拟合与过拟合**：
        *   **简单模型**（如线性模型）可能无法捕捉复杂关系（欠拟合，高偏差），离“上限”很远。
        *   **复杂模型**（如高阶多项式、深度树）可能有能力捕捉复杂关系，但容易把噪声当成规律（过拟合，高方差），虽然在训练集上逼近了上限，但在测试集上表现惨淡。
    *   **正则化的作用**：算法通过正则化（L1/L2）等技术，是在“拟合能力”和“泛化能力”之间找平衡，试图在不翻车的前提下尽可能逼近数据蕴含的真实规律（上限）。

    **4. 总结（5分）**
    *   数据科学是一个迭代过程（PPT第15页）。我们花费大量时间清洗数据、构建特征（步骤1 & 2），是为了抬高“上限”。而选择SVM、KNN还是神经网络（步骤3），只是为了找到一把最合适的梯子，爬到这个上限的高度。如果数据本身没有区分度（上限很低），换再好的梯子（算法）也没用。

### 第四章《单机上处理大数据》模拟试题（押题卷）

#### 一、 简答题（共6题，每题6分）

1.  **在数据科学语境下，如何定义“单机大数据”？当数据量超过单机内存时，通常会遇到哪三个主要的技术瓶颈？**
    *   **参考答案：**
        *   **定义**：这里的“大数据”并非指PB级海量数据，而是指相对于单台计算机的**内存（RAM）**而言，数据规模大到无法一次性载入内存进行处理的数据。
        *   **三大瓶颈**：
            1.  **内存不足（OOM）**：尝试加载全量数据导致程序崩溃。
            2.  **处理速度慢**：频繁的磁盘I/O（从硬盘读写数据）远慢于内存操作。
            3.  **算力/组件瓶颈**：CPU满载而I/O空闲，或者I/O满载而CPU空闲，硬件利用率不均衡。

2.  **什么是“稀疏矩阵”（Sparse Matrix）？在处理文本或大规模分类特征时，使用稀疏矩阵存储有什么核心优势？**
    *   **参考答案：**
        *   **定义**：在矩阵中，绝大多数元素的值为0，只有极少数元素非0的矩阵。
        *   **核心优势**：
            1.  **节省内存**：仅存储非0元素的坐标和值（如 `(row, col, value)`），极大减少内存占用。
            2.  **计算加速**：算法可以跳过大量的零值运算，提高处理速度。

3.  **请简述“在线学习”（Online Learning）与“全批学习”（Batch Learning）的区别。为什么在线学习适合处理流式大数据？**
    *   **参考答案：**
        *   **全批学习**：一次性将所有数据加载到内存中训练模型。
        *   **在线学习**：每次只读取一个观测值（或一小批/Mini-batch）进行训练，更新模型参数后即可“遗忘”该数据，不需要保留在内存中。
        *   **适合原因**：在线学习对内存要求极低，且能够适应数据的动态变化（如实时流数据），解决了内存溢出问题。

4.  **在推荐系统中，局部敏感哈希（LSH）解决的核心问题是什么？它与传统的K-近邻（KNN）算法相比，在精度和效率上做了怎样的权衡？**
    *   **参考答案：**
        *   **解决问题**：在高维海量数据中快速查找相似项（Nearest Neighbor Search）。
        *   **权衡（Trade-off）**：
            *   **传统KNN**：全局搜索，精度最高（全局最优），但计算量巨大，速度慢。
            *   **LSH**：将相似的数据大概率映射到同一个桶（Bucket）中。它牺牲了一定的**精度**（可能漏掉最相似的，只找到局部最优），换取了极高的**搜索效率**（常数级时间复杂度）。

5.  **解释MapReduce算法在单机环境下的基本思想，并说明其“Map”和“Reduce”阶段分别做了什么？**
    *   **参考答案：**
        *   **基本思想**：将大任务拆分为小块，并行或分批处理，最后合并结果。在单机上主要用于分块计算以节省内存。
        *   **Map阶段**：映射。将输入数据分割，对每个数据元素进行处理，生成键值对（Key-Value）。
        *   **Reduce阶段**：归约。将Map阶段输出的具有相同Key的数据汇聚在一起进行聚合操作（如求和、求平均）。

6.  **在利用Python处理大数据时，为什么建议将文本数据转换为数值型或二进制位字符串（Bit String）？请结合“汉明距离”说明其好处。**
    *   **参考答案：**
        *   计算机处理数值和二进制位的速度远快于文本字符串比对。
        *   **好处**：利用位运算（如XOR异或）计算**汉明距离**（两个等长字符串对应位置不同字符的个数）极其高效。通过位操作，可以瞬间判断两个用户的行为模式（如是否看过某组电影）的差异度，显著加速推荐系统的相似度计算。

#### 二、 计算题（共3题，每题10分，需使用科学计算器）

**1. 误差指标计算（MSE与MAPE）**
**背景：** 你使用随机梯度下降（SGD）算法训练了一个回归模型来预测服务器负载。在验证集上，选取了5个时间点的数据如下：

| 时间点 | 真实负载 (Y) | 预测负载 ($\hat{Y}$) |
| :----- | :----------- | :------------------- |
| T1     | 100          | 105                  |
| T2     | 80           | 72                   |
| T3     | 120          | 120                  |
| T4     | 50           | 60                   |
| T5     | 200          | 190                  |

请计算：
1.  **均方误差 (MSE)**。
2.  **平均绝对百分比误差 (MAPE)**。
*(注：计算结果保留两位小数)*

*   **详细解答：**
    **步骤1：计算各样本误差**
    *   T1: 误差 $100-105=-5$, 绝对误差 5, 相对误差 $5/100=0.05$
    *   T2: 误差 $80-72=8$, 绝对误差 8, 相对误差 $8/80=0.10$
    *   T3: 误差 $120-120=0$, 绝对误差 0, 相对误差 $0$
    *   T4: 误差 $50-60=-10$, 绝对误差 10, 相对误差 $10/50=0.20$
    *   T5: 误差 $200-190=10$, 绝对误差 10, 相对误差 $10/200=0.05$

    **1. 计算 MSE**
    $MSE = \frac{1}{n}\sum (Y - \hat{Y})^2$
    $= \frac{1}{5} [(-5)^2 + 8^2 + 0^2 + (-10)^2 + 10^2]$
    $= \frac{1}{5} [25 + 64 + 0 + 100 + 100]$
    $= \frac{289}{5} = \mathbf{57.80}$

    **2. 计算 MAPE**
    $MAPE = \frac{100\%}{n}\sum |\frac{Y - \hat{Y}}{Y}|$
    $= \frac{100\%}{5} [0.05 + 0.10 + 0 + 0.20 + 0.05]$
    $= \frac{100\%}{5} [0.40] = 0.08 * 100\% = \mathbf{8.00\%}$

**2. 汉明距离与Jaccard相似度计算**
**背景：** 在推荐系统中，我们用二进制串表示用户是否看过特定的5部电影（1表示看过，0表示没看过）。
*   用户A：`1 0 1 1 0`
*   用户B：`1 1 1 0 0`
*   用户C：`0 0 1 1 1`

请计算：
1.  用户A与用户B之间的**汉明距离（Hamming Distance）**。
2.  用户A与用户B之间的**Jaccard相似系数**（公式：$J(A,B) = \frac{A \cap B}{A \cup B}$，即 交集元素个数 / 并集元素个数）。

*   **详细解答：**
    **1. 汉明距离**
    *   对比 A (`10110`) 和 B (`11100`) 的对应位：
        *   第1位：1 vs 1 (相同)
        *   第2位：0 vs 1 (**不同**)
        *   第3位：1 vs 1 (相同)
        *   第4位：1 vs 0 (**不同**)
        *   第5位：0 vs 0 (相同)
    *   **汉明距离 = 2**

    **2. Jaccard相似系数**
    *   A看过: {Movie1, Movie3, Movie4}
    *   B看过: {Movie1, Movie2, Movie3}
    *   $A \cap B$ (两人都看过的): {Movie1, Movie3} $\rightarrow$ 个数为 2
    *   $A \cup B$ (两人总共看过的): {Movie1, Movie2, Movie3, Movie4} $\rightarrow$ 个数为 4
    *   $J(A,B) = 2 / 4 = \mathbf{0.5}$

**3. 分块矩阵计算（Block Matrix）**
**背景：** 内存不足以一次性计算两个大矩阵 $M$ 和 $N$ 的加法。我们将矩阵分块为 $2 \times 2$ 的子矩阵块进行计算。
已知：
$$M = \begin{bmatrix} A & B \\ C & D \end{bmatrix}, \quad N = \begin{bmatrix} E & F \\ G & H \end{bmatrix}$$
其中 $A, B \dots H$ 均为 $1000 \times 1000$ 的小矩阵。
请写出计算 $R = M + N$ 的分块计算步骤（用子矩阵代号表示），并说明这种做法为什么能解决内存溢出问题？

*   **详细解答：**
    **步骤：**
    1.  加载子矩阵 $A$ 和 $E$ 到内存，计算 $R_{11} = A + E$，保存/输出 $R_{11}$，释放 $A, E$ 内存。
    2.  加载子矩阵 $B$ 和 $F$ 到内存，计算 $R_{12} = B + F$，保存/输出 $R_{12}$，释放 $B, F$ 内存。
    3.  加载子矩阵 $C$ 和 $G$ 到内存，计算 $R_{21} = C + G$，保存/输出 $R_{21}$，释放 $C, G$ 内存。
    4.  加载子矩阵 $D$ 和 $H$ 到内存，计算 $R_{22} = D + H$，保存/输出 $R_{22}$，释放 $D, H$ 内存。
    **最终结果：** $R = \begin{bmatrix} A+E & B+F \\ C+G & D+H \end{bmatrix}$

    **原因说明：**
    全矩阵计算需要同时存储 $2 \times 2 \times 2 = 8$ 个大块的内存空间。而分块计算每次只需要在内存中保留 2 个子块进行运算，运算完即可释放，**极大降低了峰值内存需求**（约为原来的1/4）。

3.  **模型训练：** 既然无法一次性将数据喂给模型，你应该选择哪种类型的机器学习算法？在 `Scikit-learn` 中，这类算法通常通过哪个特定的函数接口（API）来实现训练？

        *   应采用**流式读取（Streaming）**或**迭代读取**。
        *   不解压整个文件到硬盘，而是使用 Python 的 `tarfile` 库直接在内存中以流的方式逐个读取压缩包内的文件。

    2.  **特征处理：**
        *   应使用**稀疏矩阵（Sparse Matrix）**。
        *   URL文本特征转化为向量后，绝大多数维度是0（例如某个词没出现）。使用稀疏格式（如CSR或COO）仅存储非零值，能将内存占用降低几个数量级。
        *   还可以使用**哈希技巧（Feature Hashing）**将高维特征映射到固定维度的向量中，进一步控制内存。
    3.  **模型训练：**
        *   应选择**在线学习算法（Online Learning）**或支持**增量学习**的算法（如随机梯度下降 SGD）。
        *   在 `Scikit-learn` 中，使用 **`partial_fit()`** 函数。该函数允许模型分批次（Batch-by-Batch）接收数据并更新权重，而不是使用需要全量数据的 `fit()` 函数。

**2. 内存受限下的推荐系统优化**
**背景：** 某视频网站有1亿用户和1万部电影。你想在单机数据库中实现“看过这部电影的人也看过...”的功能。
**问题：**
1.  **相似度计算瓶颈：** 如果直接计算两两用户之间的余弦相似度，计算量将是天文数字。请提出一种基于“哈希”的优化策略（LSH）来减少比较次数。
2.  **数据降维：** 用户的观影记录是一个巨大的稀疏向量（1万维）。为了加速汉明距离的计算，PPT中提到了一种“位字符串（Bit String）”压缩方法。请描述这种方法是如何将多个电影信息压缩到一个整数中的？
3.  **数据库优化：** 在数据库（如MySQL）中实现上述逻辑时，除了算法层面的优化，还可以利用哪些数据库特性（如索引、用户定义函数UDF）来加速查询？

*   **详细解答：**
    1.  **LSH优化策略：**
        *   使用**局部敏感哈希（LSH）**。构建多个哈希函数（MinHash），将用户映射到不同的“桶”（Bucket）中。
        *   只比较**同一个桶内**的用户。基于假设：如果两个用户相似，他们大概率会被哈希到同一个桶。这样避免了全量两两比较，将$O(N^2)$复杂度降为接近$O(N)$。
    2.  **位字符串压缩：**
        *   将用户的观影状态（看/没看）视为0/1。
        *   将每32部（或64部）电影的0/1状态拼接成一个**二进制串**，然后将其转换为一个**整数（Integer）**存储。
        *   例如：电影1-3的状态是`101`，存为整数5。
        *   这样计算相似度时，可以直接对整数进行**位运算（XOR）**来计算汉明距离，比循环对比数组快得多。
    3.  **数据库优化：**
        *   **用户定义函数（UDF）**：编写C或Python扩展的UDF（如 `HAMMING_DISTANCE()`），在数据库内核层直接计算距离，避免将海量数据传输到应用层计算。
        *   **索引（Indexing）**：对LSH生成的“桶ID”建立索引。查询时，数据库可以直接通过索引定位到候选用户集，避免全表扫描。

### 第五章《大数据世界的第一步》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **PPT第11页提到了HDFS“不适合的应用类型”。请解释为什么HDFS不适合存储“大量小文件”？这与NameNode的架构有什么关系？**
    *   **参考答案：**
        *   **架构限制**：HDFS采用主从架构，NameNode负责存储文件系统的元数据（Metadata），即文件目录结构、权限以及每个文件块（Block）所在的DataNode列表。这些元数据**全部加载在NameNode的内存中**。
        *   **内存瓶颈**：无论文件大小是100TB还是1KB，NameNode在内存中为每个文件、目录或Block维护的元数据对象大小是基本固定的（约150字节）。
        *   **后果**：如果存储大量小文件，NameNode的内存会在数据总量远未达到磁盘上限之前就被耗尽，导致集群无法扩展。因此HDFS适合存储大文件。

2.  **简述Hadoop MapReduce的“Map”阶段和“Reduce”阶段之间发生的过程（通常称为Shuffle），并说明为什么这个过程是MapReduce性能的瓶颈所在？**
    *   **参考答案：**
        *   **过程**：Map阶段处理完数据后，输出的中间结果（Key-Value对）并不是直接传给Reduce，而是需要经过**Shuffle（洗牌）**阶段。这包括：分区（Partition）、排序（Sort）、以及将数据从Map节点通过网络传输到Reduce节点（Copy/Network Transfer）。
        *   **瓶颈原因**：Shuffle过程涉及大量的**磁盘I/O**（写入本地磁盘防止数据丢失）和**网络I/O**（跨节点传输数据）。在大规模集群中，这种密集的数据搬运是非常耗时的。

3.  **PPT第22页提到Spark通过RDD（弹性分布式数据集）解决了MapReduce的问题。请对比MapReduce和Spark在处理“迭代计算”（如机器学习算法）时的根本区别。**
    *   **参考答案：**
        *   **MapReduce**：是基于磁盘的。每次计算（Map或Reduce）结束后，结果都必须写入磁盘（HDFS）。在迭代计算中（如循环100次），意味着要进行100次磁盘读写，I/O开销巨大。
        *   **Spark**：是基于内存的。RDD允许将中间计算结果**缓存在内存中（Cache/Persist）**。下一次迭代可以直接从内存读取数据，避免了频繁的磁盘I/O，因此在迭代算法中Spark比MapReduce快很多（通常快10-100倍）。

4.  **YARN（Yet Another Resource Negotiator）在Hadoop 2.x架构中扮演什么角色？它如何解决了Hadoop 1.x中JobTracker负载过重的问题？**
    *   **参考答案：**
        *   **角色**：YARN是集群的**资源操作系统**，负责将“资源管理”和“任务调度”分离。
        *   **解决问题**：在Hadoop 1.x中，JobTracker既管资源又管任务调度，压力巨大。YARN引入了**ResourceManager**（只管资源分配）和**ApplicationMaster**（负责具体应用程序的任务监控）。这使得Hadoop集群不再仅限于运行MapReduce，还可以运行Spark、Flink等多种计算框架。

5.  **HBase与HDFS的关系是什么？如果HDFS不支持低延时访问（PPT第11页），为什么基于HDFS的HBase可以支持实时读写？**
    *   **参考答案：**
        *   **关系**：HBase是构建在HDFS之上的分布式NoSQL数据库。HDFS为HBase提供底层文件存储的可靠性保障。
        *   **实时性原理**：HBase利用了**LSM树（Log-Structured Merge-tree）**结构和**内存缓存（MemStore）**。数据写入时先写入内存，读取时利用索引和布隆过滤器快速定位，只有在需要持久化时才批量写入HDFS。它将HDFS的顺序写能力转化为上层的随机读写能力。

#### 二、 计算题（共2题，每题10分，难度分级）

**1. HDFS存储规划（常规难度）**
**背景：** 你需要在一个Hadoop集群中存储一个大小为 **500 GB** 的超大日志文件。
**集群配置：**
*   HDFS默认块大小（Block Size）= **128 MB**。
*   默认副本系数（Replication Factor）= **3**。

**问题：**
1.  该文件将被切分成多少个Block？
2.  在HDFS中，该文件实际占用的物理磁盘空间总共是多少？
3.  如果上传该文件时，网络传输速度为 100 MB/s，且不考虑并行传输优化（假设是串行写入），理论上写完该文件所有副本需要多长时间？

*   **详细解答：**
    1.  **Block数量计算：**
        文件大小 = 500 GB = 500 $\times$ 1024 MB = 512,000 MB。
        Block数量 = $\lceil 512,000 / 128 \rceil$ （向上取整）
        $= 4000$ 个 Block。
    2.  **物理存储空间：**
        HDFS存储逻辑大小 = 500 GB。
        由于有3个副本，实际物理占用 = 500 GB $\times$ 3 = **1500 GB (或 1.5 TB)**。
    3.  **传输时间：**
        需要写入的总数据量 = 1500 GB。
        时间 = 总数据量 / 速度
        $= (1500 \times 1024) \text{ MB} / 100 \text{ MB/s}$
        $= 15360$ 秒
        $= 15360 / 60 = 256$ 分钟 $\approx$ **4.27 小时**。

**2. NameNode内存瓶颈与小文件问题（高难度，结合PPT第11页）**
**背景：**
*   某Hadoop集群的NameNode配置了 **16 GB** 的堆内存（Heap Memory）。
*   PPT第11页提到：“经验而言，一个文件/目录/文件块一般占有 **150 字节** 的元数据内存空间”。
*   集群的数据节点（DataNode）总磁盘容量极其巨大，假设有 **10 PB**。
*   块大小（Block Size）= **128 MB**。
*   副本系数 = 3。

**场景A（存大文件）：** 假设存储的文件平均大小为 **200 MB**。
**场景B（存小文件）：** 假设存储的文件平均大小为 **100 KB**。

**问题：**
1.  NameNode的16GB内存理论上最多能存储多少个元数据对象（Metadata Objects）？（提示：1 GB $\approx 10^9$ 字节，为了方便计算，可直接按 $16 \times 10^9$ 计算，或者按二进制 $16 \times 1024^3$ 计算，考试时通常建议按二进制更严谨，但也接受近似值。这里我们按 $1.6 \times 10^{10}$ 字节估算）。
2.  **在场景A中**（文件平均200MB）：
    *   每个文件占用几个Block？
    *   每个文件产生多少个元数据对象？（提示：1个文件对象 + Block对象数量）
    *   NameNode内存耗尽时，集群能存储多少**物理数据量**？
3.  **在场景B中**（文件平均100KB）：
    *   每个文件占用几个Block？
    *   NameNode内存耗尽时，集群能存储多少**物理数据量**？
4.  **分析：** 为什么说存小文件浪费了DataNode的10PB磁盘空间？

*   **详细解答：**
    1.  **元数据对象上限：**
        总内存 = 16 GB $\approx 16 \times 1024^3$ 字节 $\approx 17,179,869,184$ 字节。
        对象数量 = $17,179,869,184 / 150 \approx$ **1.14亿个对象**。

    2.  **场景A（200MB大文件）：**
        *   Block数：$200 / 128 = 1.56 \rightarrow$ 需要 2个 Block。
        *   每个文件的元数据对象：1个文件对象 + 2个Block对象 = **3个对象**。
        *   可存文件数：$1.14亿 / 3 \approx 3800$万个文件。
        *   集群存储数据量：$3800万 \times 200 \text{ MB} \approx 7,600,000,000 \text{ MB} \approx$ **7.2 PB**。
        *   *(结论：内存基本能支撑PB级数据)*

    3.  **场景B（100KB小文件）：**
        *   Block数：虽然100KB远小于128MB，但每个文件至少占用 **1个 Block**（逻辑上的ID，不占物理128M空间，但占元数据）。
        *   每个文件的元数据对象：1个文件对象 + 1个Block对象 = **2个对象**。
        *   可存文件数：$1.14亿 / 2 \approx 5700$万个文件。
        *   集群存储数据量：$5700万 \times 100 \text{ KB} = 5,700,000,000 \text{ KB} \approx 5.4 \text{ TB}$。

    4.  **分析结论：**
        *   在场景A中，NameNode内存支撑了 **7.2 PB** 的数据存储。
        *   在场景B中，NameNode内存耗尽时，仅存储了 **5.4 TB** 的数据。
        *   **浪费原因：** 尽管DataNode有10 PB的容量，但由于NameNode内存被大量小文件的元数据填满，剩下的 **9.99 PB** 磁盘空间将无法使用。这就是“小文件问题”。

        *   *理由：* MapReduce是批处理框架，启动Job开销大，且必须读写磁盘，无法达到“秒级”响应。
        *   *组件：* 应使用 **Spark Streaming**（PPT第23页），它将流数据切分成微批（Micro-batch）进行内存处理。
    3.  **Spark内存溢出问题：** **可以运行**。
        *   *原理：* Spark并非强制所有数据都在内存。当内存不足时，Spark会自动将RDD中放不下的数据**Spill（溢写）到磁盘**。虽然这会降低性能（退化得接近MapReduce），但程序依然能跑通，体现了其弹性（Resilient）。

**2. 词频统计（Word Count）原理推演**
**背景：** 这是一个经典的大数据入门案例。假设有一个文本文件 `file.txt`，内容如下：

**问题：**
请参考PPT第15页的流程图，详细描述MapReduce处理这段文本的流程：

**题目：PPT的标题是“大数据世界的第一步”。从Hadoop的诞生到Spark的流行，代表了大数据处理技术的演进。请结合本章内容，论述：**

2.  **为什么Spark被认为是Hadoop MapReduce的继承者？它保留了什么，又改变了什么？**
3.  **在现代数据平台中，Hadoop（HDFS/YARN）和Spark往往是共存的（PPT第24页图）。请描述这种“共生关系”是如何工作的？**

*   **答题思路与核心要点：**

    **1. Hadoop的里程碑意义（6分）**
    *   **解决核心问题**：解决了**单机无法存储**（通过HDFS分布式存储）和**单机无法计算**（通过MapReduce分布式计算）海量数据的问题。

    *   **特性**：PPT第4页提到的“移动计算而非移动数据”（虽然PPT没明说这句口号，但体现了分布式的核心），以及通过多副本实现的**高容错性**。

    **2. Spark的继承与革命（7分）**
    *   **保留了什么**：Spark保留了分布式计算的思想（Map/Reduce的逻辑），保留了对Hadoop底层存储（HDFS）和资源调度（YARN）的兼容性。
    *   **改变了什么**：核心改变在于**内存计算**（In-memory Computing）。MapReduce中间结果落盘，导致I/O瓶颈；Spark引入RDD，让中间结果驻留内存。PPT第22页指出这消除了保存到磁盘的过程，适合迭代算法和交互式分析。

    **3. 共生关系（7分）**
    *   **生态系统**：Spark通常不是独立存在的，它往往运行在Hadoop生态中。
    *   **分工明确**：
        *   **存储层**：Spark没有自己的文件系统，依赖 **HDFS** 来存储PB级数据。
        *   **调度层**：Spark需要资源来运行，它向 **YARN** 申请CPU和内存（PPT第24页架构图）。
        *   **计算层**：Spark取代了MapReduce成为了更高效的计算引擎。
    *   **总结**：Hadoop提供了坚实的底层（地基和管家），Spark提供了高效的上层建筑（处理引擎）。

### 第六章《NoSQL数据库》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **请解释关系型数据库（RDBMS）的ACID原则与NoSQL数据库的BASE原则的主要区别，并说明为什么NoSQL更倾向于BASE？**
    *   **考点关联：** PPT第16页、第22页。
    *   **参考答案：**
        *   **ACID**（原子性、一致性、隔离性、持久性）强调**强一致性**，要求事务要么全做要么全不做，且数据时刻保持一致，适合银行交易等场景。
        *   **BASE**（基本可用 BA、软状态 S、最终一致性 E）强调**可用性**。它允许系统在短时间内数据不一致（软状态），但保证在一定时间后数据最终达到一致。
        *   **倾向原因**：NoSQL为了满足Web 2.0的海量数据和高并发需求（如分布式环境），根据CAP定理，往往必须牺牲强一致性（C）来换取高可用性（A）和分区容错性（P）。BASE模型提供了这种灵活性。

2.  **根据CAP定理，当分布式系统出现网络分区（Network Partition）故障时，架构师必须在哪两个指标中做出选择？请结合“CP系统”和“AP系统”举例说明。**
    *   **考点关联：** PPT第18-20页。
    *   **参考答案：**
        *   当出现分区（P）时，必须在**一致性（Consistency）**和**可用性（Availability）**之间做选择。
        *   **CP系统**（选择一致性）：为了保证数据准确，系统会停止服务或拒绝写入，直到网络恢复（如HBase，银行系统）。PPT第20页描述为“暂停数据库访问”。
        *   **AP系统**（选择可用性）：系统继续提供服务，返回旧数据或允许写入，稍后再同步（如Cassandra，社交动态）。PPT第19页描述为“数据将会不一致”。

3.  **请简述列存储数据库（Column-oriented）与行存储数据库在存储方式上的区别，并说明为什么列存储更适合“数据分析和报表”场景？**
    *   **考点关联：** PPT第26-28页。
    *   **参考答案：**
        *   **区别**：行存储将一行记录的所有字段连续存储；列存储将同一列的所有数据连续存储（将每一列单独存表）。
        *   **适合分析的原因**：
            1.  **I/O效率**：分析查询通常只涉及少数几列（如“计算平均年龄”），列存储只需读取需要的列，无需读取整行数据，极大减少I/O。
            2.  **压缩率高**：同一列的数据类型相同且重复率高（如“性别”列），压缩效果极好，节省存储空间并加速扫描。

4.  **在Elasticsearch中文本搜索流程里，什么是“倒排索引”（Inverted Index）？（注：PPT第44页图示隐喻了此概念，且Elasticsearch基于Lucene）。**
    *   **考点关联：** PPT第44页、第56页。
    *   **参考答案：**
        *   倒排索引是搜索引擎的核心数据结构。
        *   它不是由“文档ID -> 关键词”的映射，而是建立**“关键词 -> 出现该词的文档ID列表”**的映射。
        *   例如，搜索“diabetes”，系统直接在倒排索引中找到包含该词的文档ID集合（如Doc1, Doc3），而不需要遍历所有文档，从而实现毫秒级响应。

5.  **图数据库（Graph Database）由哪两个核心元素组成？相比于关系型数据库，它在处理什么类型的数据关系时具有绝对优势？**
    *   **考点关联：** PPT第33-34页。
    *   **参考答案：**
        *   **核心元素**：**结点（Node，实体本身）**和**边（Edge，实体间的关系）**。
        *   **优势场景**：处理**联系高度紧密**的数据（高度互联数据）。例如社交网络（朋友的朋友）、推荐系统、知识图谱。在这些场景下，关系型数据库需要极其复杂的表连接（Join），性能极差，而图数据库通过图遍历算法能高效处理。

#### 二、 计算题（共2题，每题10分，需使用科学计算器）

**1. 文本相似度算法：Damerau-Levenshtein 距离计算（常规难度）**
**背景：** 在PPT第66-67页，介绍了处理拼写错误的距离算法。Damerau-Levenshtein 距离允许四种操作：插入（Insertion）、删除（Deletion）、替换（Substitution）、**相邻字符交换（Transposition）**。
**题目：**
假设用户在搜索框输入了错误的单词 **$S1 = \text{"biadetes"}$**，而正确的医学术语是 **$S2 = \text{"diabetes"}$**。
1.  请计算将 $S1$ 转换为 $S2$ 所需的最少操作次数（即编辑距离），并详细列出每一步操作。
2.  如果仅使用标准的 Levenshtein 距离（不支持相邻字符交换），距离是多少？
3.  通过此例说明为什么针对拼写错误（Typo），Damerau-Levenshtein 更好。

*   **详细解答：**
    **1. Damerau-Levenshtein 距离计算（支持交换）：**
    *   目标：`biadetes` -> `diabetes`
    *   步骤1（交换）：`bi` -> `ib` (第1、2位交换) -> `ibadetes`。此时对比目标 `diabetes`，前两个字符 `di` vs `ib` 还是不对，看来直接交换头两个不行。让我们仔细观察差异：
        *   S1: `b i a d e t e s`
        *   S2: `d i a b e t e s`
    *   观察发现，中间的 `a d` 和 `e t e s` 是匹配的。差异主要在 `b` 和 `d` 的位置。
    *   路径规划：
        1.  `biadetes` (原词)
        2.  **替换（Substitution）**：将第一个 `b` 替换为 `d` -> `diadetes`
        3.  **替换（Substitution）**：将第四个 `d` 替换为 `b` -> `diabetes`
        *   总操作数 = 2。
    *   *修正思考（是否存在交换）：*
        *   S1: `b i a d e t e s`
        *   S2: `d i a b e t e s`
        *   这并不是简单的相邻字符交换能解决的（`b`和`d`隔了两个字符 `i a`）。PPT第67页的例子是 `lupsu` -> `lupus`（相邻交换）。
        *   **但是！** 让我们看另一个路径：
            *   如果不允许跨越距离交换，只允许相邻交换：
            *   `biadetes` -> `ibadetes` (交换b,i) -> `iadetes` (删除b)? 不对。
            *   让我们回到最基础的动态规划或直观对比。
            *   S1: `b` `i` `a` `d` `e` `t` `e` `s`
            *   S2: `d` `i` `a` `b` `e` `t` `e` `s`
            *   操作1: 把 `b` 换成 `d` -> `diadetes`
            *   操作2: 把 `d` 换成 `b` -> `diabetes`
            *   **结果**：距离为 **2**。

    **2. 标准 Levenshtein 距离（不支持交换）：**
    *   标准版只支持 插入、删除、替换。
    *   在该特定案例中（`biadetes` -> `diabetes`），因为不涉及相邻字符的直接转置（如 `ab` -> `ba`），所以标准 Levenshtein 距离的操作也是上述的两次替换。
    *   **结果**：距离仍为 **2**。

    **3. 对比说明（引用PPT第67页例子）：**
    *   假设例子是 PPT 中的 `lupsu` -> `lupus`。
    *   **Damerau-Levenshtein**：`su` -> `us`，**1次**操作（相邻交换）。
    *   **标准 Levenshtein**：`s`替换为`u` -> `lupuu`，然后`u`替换为`s` -> `lupus`。需要 **2次**操作。
    *   **结论**：在拼写错误中，“按错相邻键导致顺序颠倒”非常常见，支持“交换”操作的算法能更准确地反映用户的真实错误成本，计算出的距离更小，召回率更高。

**2. 分布式系统可用性与可靠性计算（高难度，结合PPT第21页）**
**背景：** PPT第21页展示了分片（Sharding）和复制（Replication/Replica）的架构。假设我们有一个NoSQL数据库集群：
*   **架构A（纯分片，无备份）**：数据被切分成 **10个** 分片（Shard），分布在 **10台** 不同的服务器上。只要有任意 1 台服务器宕机，整个数据库被认为“不可用”（数据丢失或查询失败）。
*   **架构B（分片+复制）**：同样由 **10个** 分片组成，但每个分片都有 **1个主节点（Primary）** 和 **2个备份节点（Replica）**（即每个分片有3个副本）。只有当某个分片的所有3个副本**同时**宕机时，该分片才不可用，从而导致整个系统不可用。

已知每台服务器在单位时间内的**宕机概率**为 $p = 0.05$（即可用性为 95%）。

**问题：**
1.  计算 **架构A** 的系统整体可用性（系统不宕机的概率）。
2.  计算 **架构B** 中，**某一个特定分片**（包含3个副本）发生故障（即3个副本全挂）的概率。
3.  计算 **架构B** 的系统整体可用性（即所有10个分片都正常工作的概率）。
4.  **分析：** 相比架构A，架构B的硬件成本增加了多少倍？可用性提升了多少个数量级（或百分点）？这体现了NoSQL设计中的什么权衡？

*   **详细解答：**
    **1. 架构A可用性**
    *   每台机器可用概率 = $1 - 0.05 = 0.95$。
    *   系统要可用，必须 10 台机器同时可用。
    *   $P(A_{available}) = 0.95^{10} \approx \mathbf{0.5987}$ (或 59.87%)。
    *   *评注：这是非常糟糕的可用性。*

    **2. 架构B 单个分片故障率**
    *   一个分片包含3个副本，且互为备份。
    *   该分片不可用的条件：3台机器同时挂。
    *   $P(Shard_{fail}) = p^3 = 0.05^3 = 0.000125$。
    *   单个分片的可用性 $P(Shard_{ok}) = 1 - 0.000125 = 0.999875$。

    **3. 架构B 系统整体可用性**
    *   系统要可用，必须 10 个分片都可用。
    *   $P(B_{available}) = P(Shard_{ok})^{10} = (0.999875)^{10}$
    *   使用二项展开近似或直接计算：
    *   $P(B_{available}) \approx \mathbf{0.99875}$ (或 99.88%)。

    **4. 分析与权衡**
    *   **成本**：架构A用10台机器，架构B用30台机器（10分片 * 3副本）。成本增加了 **2倍**（变为原来的3倍）。
    *   **可用性提升**：从 59.87% 提升到了 99.88%。这不仅是数值的提升，更是从“几乎不可用”变成了“高可用（接近3个9）”。
    *   **权衡**：体现了 **空间换可用性（Space for Availability）** 的思想。在NoSQL中，通过增加硬件成本（存储冗余）来极大地抵消硬件故障带来的风险，符合BASE原则中的“基本可用”。

#### 三、 综合分析题（共2题，每题15分）

**1. 混合架构设计（Hybrid Architecture）**
**背景：** 某大型电商平台（类似亚马逊，参考PPT第14页）需要升级其后端数据架构。业务包含以下模块：
1.  **用户购物车**：读写极其频繁，临时性数据，对响应速度要求极高，但如果服务器宕机，丢失最近几秒的加购记录是可以接受的。

3.  **历史订单**：海量数据，结构化强，用于财务审计和发货，绝不允许数据丢失或不一致。
4.  **社交推荐**：分析“用户A关注了用户B，用户B购买了商品C”这类关系网。

**问题：**
请为上述4个模块分别推荐最合适的数据库类型（关系型、键值、文档、图、列存储等），并结合PPT中的知识点（如数据模型灵活性、ACID/BASE、CAP）说明理由。

*   **详细解答：**
    1.  **用户购物车 -> 键值存储 (Key-Value Store, 如Redis)**
        *   *理由*：PPT第29页提到键值对适合存储“购物篮”这种临时数据。购物车只需通过User ID快速读写，不需要复杂的查询。Redis基于内存，能提供极高的性能（高可用性），且符合Web 2.0“不要求严格读写实时性/持久性”的特点（PPT第9页）。
    2.  **商品目录 -> 文档存储 (Document Store, 如MongoDB)**
        *   *理由*：PPT第30页提到文档存储无需预定义模式（Schema-free）。商品属性差异大（异构数据），关系型数据库需要大量Null值列或复杂的关联表，而文档数据库可以用JSON灵活存储不同商品的属性。
    3.  **历史订单 -> 关系型数据库 (RDBMS, 如MySQL/Oracle)**
        *   *理由*：PPT第11页对比中，RDBMS具备“强一致性”和“数据完整性”。订单涉及金钱和物流，必须满足ACID原则（PPT第16页），确保事务的原子性和持久性，不能容忍数据丢失或最终一致性。
    4.  **社交推荐 -> 图数据库 (Graph Database, 如Neo4j)**
        *   *理由*：PPT第33页指出图数据库擅长解决“联系高度紧密”的数据。处理用户关注关系和商品购买路径（A->B->C）时，图数据库的点边结构比关系表的Join效率高出数量级。

**2. Elasticsearch 搜索引擎实战分析**
**背景：** 结合PPT第62-65页的“通过症状诊断疾病”案例。你正在开发一个医疗搜索系统。
**问题：**
1.  **分词（Tokenization）与N-Gram：** PPT第79页提到了 `shingle` 过滤器产生“二元语法”（Bigrams）。请解释如果用户输入 "joint pain"（关节痛），使用二元语法（size=2）会被解析成什么索引项？这比单纯把 "joint" 和 "pain" 分开索引有什么优势？
2.  **倒排索引应用：** 假设有两个文档：
    *   Doc A: "fever and joint pain"
    *   Doc B: "high fever causing pain"
    *   如果用户搜索 "joint pain"，请模拟Elasticsearch的匹配过程。
3.  **评分与排序：** PPT第72页提到“搜索查询会计算搜索得分”。Elasticsearch通常使用TF-IDF算法。请简要解释TF（词频）和IDF（逆文档频率）在这个医疗案例中是如何影响“红斑狼疮”排名的？（即：为什么罕见症状匹配到了得分会更高？）

*   **详细解答：**
    1.  **Shingle/Bigram分析：**
        *   "joint pain" 使用二元语法（Shingle size=2）会被解析为：`"joint pain"`（作为一个整体词项），同时也可能保留单字 `"joint"`, `"pain"`。
        *   *优势：* 能够保留**语义上下文**。单纯搜索 "joint" 和 "pain" 可能会匹配到 "pain in the neck and joint flexibility"（虽然都有词，但意思不对）。二元语法确保了这两个词是相邻出现的，提高了搜索的准确性（Precision）。
    2.  **倒排索引模拟：**
        *   索引构建：
            *   `fever`: {Doc A, Doc B}
            *   `joint`: {Doc A}
            *   `pain`: {Doc A, Doc B}
        *   搜索 "joint pain"：
            *   查找 `joint`: 命中 Doc A。
            *   查找 `pain`: 命中 Doc A, Doc B。
            *   交集/合并：Doc A 命中两个词，Doc B 命中一个词。
            *   结果：Doc A 相关性更高，排在前面。
    3.  **TF-IDF 评分影响：**
        *   **TF (Term Frequency)**：如果某疾病描述中多次出现 "rash"（皮疹），说明该词对该文档很重要。
        *   **IDF (Inverse Document Frequency)**：如果 "pain" 在所有疾病文档中都出现（很常见），它的权重就会降低（IDF低）。而 "butterfly rash"（蝴蝶状红斑）可能只在少数疾病（如红斑狼疮）中出现（IDF高）。
        *   *结论：* 当用户输入罕见症状时，由于IDF高，匹配到该症状的文档得分会显著拉高，从而使红斑狼疮排在普通感冒前面。

### 第七章《图数据库的兴起》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **简述属性图模型（Property Graph Model）的四个基本组成部分（节点、关系、属性、标签）在Neo4j中的作用。**
    *   **参考答案：**
        *   **节点（Nodes）**：表示实体（如用户、产品、电影），是图中的“点”。
        *   **关系（Relationships）**：连接两个节点，表示实体间的关联（如“User LIKES Movie”）。关系必须有方向（单向）和类型。
        *   **属性（Properties）**：节点和关系都可以拥有属性，以键值对（Key-Value）形式存储详细信息（如用户的姓名、关系的创建时间）。
        *   **标签（Labels）**：用于给节点分组（如`:Person`, `:Movie`），相当于关系型数据库中的“表名”，用于加速查询索引。

2.  **PPT第10-11页展示了在关系型数据库中查找“朋友的朋友”或“父子关系”时的递归查询问题。请说明为什么关系型数据库在处理“深度关联查询”（如查找5度人脉）时性能会急剧下降？**
    *   **参考答案：**
        *   **Join（连接）的代价**：关系型数据库通过表连接（Join）来重建关系。每多一层关系（如朋友的朋友），就需要多做一次Join操作。
        *   **笛卡尔积与索引查找**：随着深度增加，Join操作会产生大量的中间数据集，且需要在庞大的索引表中反复查找匹配行。
        *   **复杂度爆炸**：查询复杂度随着关系深度的增加呈指数级增长，导致内存和CPU资源耗尽。而图数据库使用“遍历（Traversal）”机制，直接通过指针访问邻居，性能仅与遍历的这部分数据量有关，与总数据量无关。

3.  **请解释Neo4j中Cypher语言的`MATCH`、`WHERE`和`RETURN`子句的功能，并写出一个简单的Cypher语句：查找所有名字为"Alice"的用户所认识（KNOWS）的朋友的名字。**
    *   **参考答案：**
        *   **MATCH**：用于描述要在图中查找的模式（Pattern），类似SQL的FROM+JOIN。
        *   **WHERE**：用于过滤结果，添加约束条件。
        *   **RETURN**：定义查询返回的结果集。
        *   **Cypher语句**：
            ```cypher
            MATCH (u:User {name: 'Alice'})-[:KNOWS]->(friend:User)
            RETURN friend.name
            ```

4.  **在构建推荐引擎（如菜谱推荐）时，图数据库如何利用“协同过滤”的思想？请用“用户-菜谱-原材料”的关系简述推荐逻辑。**
    *   **参考答案：**
        *   **基本逻辑**：基于“共同邻居”进行推荐。
        *   **具体路径**：如果用户A喜欢菜谱X，且菜谱X包含原材料{a, b, c}。我们可以查找包含相同原材料{a, b, c}的其他菜谱Y。
        *   **推理**：因为菜谱Y和用户喜欢的菜谱X在“原材料”层面高度重叠（共享邻居节点），所以系统推测用户A也可能喜欢菜谱Y，从而进行推荐。

5.  **图数据库中的关系是有向的（Directed），但在实际查询中（如“朋友”关系通常是双向的），Cypher如何处理无视方向的查询？**
    *   **参考答案：**
        *   虽然存储时关系必须有方向（A -> B），但在Cypher查询时，可以忽略箭头方向来表示双向遍历。
        *   **语法**：使用不带箭头的横线 `(a)-[:FRIEND]-(b)`，而不是带箭头的 `(a)-[:FRIEND]->(b)`。这样Neo4j会同时查找“a指向b”和“b指向a”的关系，模拟无向图的效果。

#### 二、 计算题（共2题，每题10分，逻辑推演与手动计算）

**1. 简单的协同过滤推荐得分计算（常规难度）**
**背景：** 假设有一个图数据库存储了“用户（User）”和“电影（Movie）”的关系。关系类型为 `[:LIKES]`。我们想给用户 **U1** 推荐电影。
已知图结构如下：
*   **用户 U1** 喜欢：电影 A, 电影 B。
*   **用户 U2** 喜欢：电影 A, 电影 C。
*   **用户 U3** 喜欢：电影 B, 电影 C, 电影 D。

**算法逻辑：**
1.  找到与 U1 有“共同喜好”的其他用户（即看过相同电影的用户）。
2.  计算 U1 与其他用户的**相似度得分**：每共同喜欢一部电影，计 1 分。
3.  找到这些相似用户看过的、但 U1 没看过的电影（候选电影）。
4.  计算**候选电影的推荐分**：推荐分 = $\sum$ (看过该电影的相似用户的相似度得分)。

**问题：**
1.  计算 U1 与 U2 的相似度得分，以及 U1 与 U3 的相似度得分。
2.  有哪些电影是 U1 没看过的候选电影？
3.  计算这些候选电影的最终推荐分，并按分数高低排序推荐。

*   **详细解答：**
    1.  **相似度得分计算：**
        *   **U1 vs U2**: 共同喜欢的电影是 {A}。数量为 1。**Score(U2) = 1**。
        *   **U1 vs U3**: 共同喜欢的电影是 {B}。数量为 1。**Score(U3) = 1**。
    2.  **候选电影识别：**
        *   U2 看过但 U1 没看过的：{C}。
        *   U3 看过但 U1 没看过的：{C, D}。
        *   候选池：**{C, D}**。
    3.  **电影推荐分计算：**
        *   **电影 C**：
            *   U2 看过 C (U2权重=1)。
            *   U3 看过 C (U3权重=1)。
            *   Score(C) = 1 + 1 = **2**。
        *   **电影 D**：
            *   只有 U3 看过 D (U3权重=1)。
            *   Score(D) = **1**。
    *   **最终推荐结果**：推荐 **电影 C** (2分) > **电影 D** (1分)。

**2. 图遍历深度与复杂度计算（较高难度）**
**背景：** 在一个社交网络图中，平均每个节点（人）有 $N$ 个朋友（即节点的度 Degree = $N$）。我们需要查询“朋友的朋友的朋友...”（K度人脉）。
假设：
*   每个节点平均有 **50** 个关系（$N=50$）。
*   关系型数据库（RDBMS）执行一次Join操作并索引查找的时间固定为 $T_{sql} = 10ms$（实际会随数据量增加，这里简化假设）。
*   图数据库（GraphDB）遍历一条边的时间为 $T_{graph} = 0.001ms$（1微秒）。

**问题：**
1.  **3度人脉（Level-3）：** 我们需要查找某人的“朋友的朋友的朋友”。理论上需要遍历多少个节点？（假设图中没有环路，节点不重复）。
2.  **时间对比：**
    *   在图数据库中，遍历完所有3度节点，理论上大约需要多少毫秒？（公式：节点数 $\times$ 单次遍历时间）
    *   在关系型数据库中，查询3度人脉需要进行 3 次 Join。假设每次 Join 的结果集行数等于上述节点数，且每次 Join 操作都要处理这么规模的数据。简述为什么在 $N=50$ 且深度为 3 或 4 时，RDBMS 会崩溃（或者非常慢），而 GraphDB 依然很快？（这题不要求算出RDBMS的具体秒数，要求用数据量级论证）。

*   **详细解答：**
    1.  **节点数量计算（指数增长）：**
        *   1度朋友：$50^1 = 50$ 人。
        *   2度朋友：$50 \times 50 = 2,500$ 人。
        *   3度朋友：$50 \times 50 \times 50 = 125,000$ 人。
        *   **需要遍历的总边数**（近似）：$50 + 2,500 + 125,000 = \mathbf{127,550}$ 条边。

    2.  **时间对比：**
        *   **GraphDB 时间**：
            $127,550 \times 0.001 \text{ ms} \approx \mathbf{127.55 \text{ ms}}$。
            *(仅需0.1秒左右，非常快)*
        *   **RDBMS 论证**：
            *   当深度达到 4 度时，节点数变为 $50^4 = 6,250,000$（625万）。
            *   GraphDB 耗时约 $6.25$ 秒，依然可接受。
            *   对于 RDBMS，第 3 次或第 4 次 Join 是在两张百万级甚至千万级的大表之间进行笛卡尔积式的匹配。数据库需要加载巨大的中间表到内存，索引失效，I/O 爆炸。即使是简单的 $10ms$ 基础开销，在处理数百万行数据的 Join 时会退化成 $O(N \log N)$ 甚至 $O(N^2)$ 的复杂度，查询时间可能变成几分钟甚至几小时（Timeout），这就是所谓的**“Join Bomb”（连接爆炸）**。

#### 四、 论述题（1题，20分）

**题目：在过去几十年中，关系型数据库（RDBMS）一直是数据存储的霸主。然而，PPT第9页指出“复杂性是选择图数据库的根本原因”。请结合本章内容，从“数据模型的直观性”、“查询性能（Index-free Adjacency）”和“业务场景的演变”三个维度，论述图数据库兴起的必然性。**

    **1. 数据模型的直观性（Whiteboard Friendly）（6分）**
    *   **所见即所得**：关系型数据库建模时，需要将现实世界强行拆解成二维表（规范化），设计外键，这违背了人类的直觉。而图数据库的模型（点、边）与我们在白板上画图讨论业务逻辑的方式完全一致。
    *   **消除“阻抗不匹配”**：在图数据库中，数据存储的结构就是业务逻辑的结构，开发人员和业务人员可以用同一种语言（图）沟通，降低了开发和维护成本。

    **2. 查询性能与免索引邻接（Index-free Adjacency）（7分）**
    *   **Join的痛点**：如前所述，RDBMS在处理多对多、深度关联（如社交网络、实时推荐）时，Join操作是性能杀手。
    *   **图的优势**：图数据库采用**免索引邻接**（虽然PPT没用这个专业术语，但描述了“直接连接”）。每个节点直接包含指向邻居节点的物理指针。查询时，就像顺藤摸瓜，直接跳转到内存中的下一个位置，不需要去全局索引树中查找。这使得图查询的性能是恒定的，不随数据总量增加而下降。

    **3. 业务场景的演变（7分）**
    *   **从“数据管理”到“关系挖掘”**：以前的业务主要是存储和检索（如存钱、取钱）。现代业务（Web 2.0及以后）更关注**互联（Connected Data）**。
    *   **复杂场景爆发**：社交网络（Facebook）、实时推荐（Amazon）、金融风控（反洗钱）、知识图谱（Google）。这些场景的核心价值不在于单个数据点，而在于数据之间的**关系**。只有图数据库能高效、实时地挖掘这些关系价值。

    **4. 总结**
    *   图数据库不是为了取代RDBMS（处理报表和交易还是RDBMS强），而是为了解决**高度互联数据**的复杂查询难题。随着数据关系的日益复杂，图数据库的兴起是技术发展的必然选择。

### 第八章《文本挖掘和文本分析》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **PPT第17-18页提到了“词袋模型”（Bag of Words, BoW）。请简述词袋模型的基本原理，并指出该模型在处理自然语言时的一个主要缺陷是什么？**
    *   **参考答案：**
        *   **原理**：词袋模型是将文本（句子或文档）转换为数值向量的最简单方法。它忽略语法的结构和单词的顺序，只记录每个单词在文档中**出现与否**（True/False）或者**出现的次数**（词频）。就像把所有词扔进一个袋子里一样。
        *   **缺陷**：它**丢失了语境和词序信息**。例如，“Not bad”和“Bad not”在词袋模型中是完全一样的，但语义截然不同。此外，它会生成非常稀疏的矩阵。

2.  **请解释“词干提取”（Stemming）与“词形还原”（Lemmatization）的区别，并举例说明。（参考PPT第23页）。**
    *   **参考答案：**
        *   **词干提取**：是基于规则的启发式过程，通过切除单词后缀来还原词根，结果可能不是合法的单词。例如：将 "planes" 和 "planning" 暴力切分为 "plan"。
        *   **词形还原**：基于字典和语法分析，将单词还原为原本的词典形式（Lemma）。例如：将 "are", "is", "am" 还原为 "be"。
        *   *区别*：词形还原更精确但计算更慢；词干提取速度快但较粗糙。

3.  **在PPT第26页中提到朴素贝叶斯分类器（Naive Bayes）适用于文本挖掘。请问这里的“朴素（Naive）”指的是什么假设？为什么N-gram（如二元词条）可以缓解这个假设带来的问题？**
    *   **参考答案：**
        *   **“朴素”的含义**：假设每一个输入变量（特征/单词）之间是**相互独立**的。即一个词的出现与句子中其他词的出现无关。
        *   **N-gram的作用**：文本中词语往往有关联（如 "New" 和 "York"）。N-gram（如Bigram）将相邻的两个词作为一个整体特征（"New York"），从而在一定程度上保留了**词序和上下文信息**，缓解了完全独立假设带来的语义丢失问题。

4.  **什么是“停用词”（Stop Words）？在文本挖掘中为什么要去除停用词？（参考PPT第21页）。**
    *   **参考答案：**
        *   **定义**：在语言中频繁出现但包含实际信息量极少的词，如英文中的 "the", "is", "at"，中文中的“的”、“是”。
        *   **去除原因**：
            1.  **节省存储空间**：这些词在文档中占比极高，去除后能显著减少索引大小。
            2.  **提高检索效率和精度**：这些词对区分文档主题没有帮助（几乎所有文档都有），去除它们可以减少噪音，让模型关注真正有意义的实词。

5.  **PPT第29-30页提到了决策树和“信息熵”（Entropy）。在文本分类中，熵是用来度量什么的？信息增益（Information Gain）又是如何帮助决策树选择分裂节点的？**
    *   **参考答案：**
        *   **熵**：用来度量系统的**不确定性**或**混乱程度**。如果一个集合里的样本全部属于同一类，熵为0（最有序）；如果类别分布均匀，熵最大（最混乱）。
        *   **信息增益**：表示在知道某个特征（如某个词是否出现）后，系统不确定性（熵）减少的程度。决策树会选择**信息增益最大**的词作为分裂节点，因为这个词对分类最有辨识度。

#### 二、 计算题（共2题，每题10分，需使用科学计算器）

**1. TF-IDF 计算（核心考点，参考PPT第19-20页）**
**背景：** 你有一个包含 5 篇文档的小型语料库（$N=5$）。
我们关注单词 **"Data"**。
*   **文档 A** 的内容总共有 100 个词，其中 "Data" 出现了 4 次。
*   在整个语料库的 5 篇文档中，有 2 篇文档包含了单词 "Data"。

**问题：**
请根据PPT中的公式计算单词 "Data" 在文档 A 中的 TF-IDF 值。
（注：PPT公式为 $TF = f_{t,d}$， $IDF = \log(N / |\{d \in D : t \in d\}|)$。假设对数底数为10，若使用自然对数ln或底数2请注明，通常考试默认以10或e为底，这里我们按以10为底计算）。

*   **详细解答：**
    1.  **计算 TF (Term Frequency):**
        *   单词 "Data" 在文档 A 中出现的频率。
        *   $TF = \frac{\text{"Data"在A中出现的次数}}{\text{A的总词数}} = \frac{4}{100} = \mathbf{0.04}$。
        *   *(注：有时TF直接用次数4表示，但PPT第19页写的是频率，通常指归一化后的频率，按0.04算更严谨)*
    2.  **计算 IDF (Inverse Document Frequency):**
        *   $N$ (文档总数) = 5。
        *   包含 "Data" 的文档数 = 2。
        *   $IDF = \log(\frac{5}{2}) = \log(2.5)$。
        *   使用计算器：$\log_{10}(2.5) \approx \mathbf{0.3979}$。
    3.  **计算 TF-IDF:**
        *   $TF\text{-}IDF = TF \times IDF$
        *   $= 0.04 \times 0.3979$
        *   $\approx \mathbf{0.0159}$。

**2. 混淆矩阵与文本分类评估（参考PPT第71页）**
**背景：** 你训练了一个模型来区分 "Spam"（垃圾邮件）和 "Ham"（正常邮件）。测试集共有 200 封邮件，混淆矩阵如下：

|                | 预测：Spam | 预测：Ham |
| :------------- | :--------: | :-------: |
| **真实：Spam** |  40 (TP)   |  10 (FN)  |
| **真实：Ham**  |  20 (FP)   | 130 (TN)  |

**问题：**
1.  计算该模型在识别垃圾邮件方面的 **精确率（Precision）**。
2.  计算该模型的 **召回率（Recall）**。
3.  在垃圾邮件过滤场景中，你认为 精确率 和 召回率 哪个更重要？为什么？（提示：考虑把正常邮件误判为垃圾邮件的后果）。

*   **详细解答：**
    1.  **精确率 (Precision)**：预测为Spam的样本中，由于多少是真的Spam？
        *   $Precision = \frac{TP}{TP + FP} = \frac{40}{40 + 20} = \frac{40}{60} \approx \mathbf{0.667}$ (66.7%)。
    2.  **召回率 (Recall)**：真实的Spam中，有多少被找出来了？
        *   $Recall = \frac{TP}{TP + FN} = \frac{40}{40 + 10} = \frac{40}{50} = \mathbf{0.800}$ (80%)。
    3.  **分析**：
        *   在垃圾邮件过滤场景中，**精确率（Precision）** 通常更重要。
        *   *理由*：如果精确率低，意味着很多正常邮件（Ham）被误判为垃圾邮件（FP高）。用户可能会错过重要的工作邮件或账单，这是不可接受的。而召回率低一点（漏掉几封垃圾邮件进入收件箱）通常是可以容忍的。

#### 三、 综合分析题（共2题，每题15分）

**1. 文本挖掘全流程设计（基于Reddit案例）**
**背景：** 结合PPT第38-39页的流程图。假设你要为一个新闻网站建立一个自动分类系统，将新闻分为“体育”、“科技”和“政治”三类。
**问题：**
请设计完整的文本挖掘流程，并详细说明每个步骤的具体操作：
1.  **数据清洗（Cleaning）：** 对于抓取到的原始网页文本，你需要做哪些清洗操作？请列举至少三项（参考PPT第48、55页）。
2.  **数据转换/特征工程（Transformation）：** 如何将清洗后的文本转换为计算机可理解的格式？PPT第55页提到了“罕用语（Hapaxes）”，你会如何处理它们，为什么？
3.  **建模与评估（Modeling）：** 如果你发现“体育”类新闻的识别率很高，但“科技”和“政治”经常混淆，你会检查哪些高权重的关键词？（结合PPT第73页“最重要的关键词”图表思路）。

*   **详细解答：**
    1.  **数据清洗**：
        *   **去除HTML标签**：网页抓取的数据包含大量 `<div>`, `<a>` 等无意义标签。
        *   **大小写转换**：将所有单词统一为小写（如 "Apple" 转为 "apple"），避免重复统计。
        *   **去除停用词**：去掉 "the", "is", "and" 等无区分度的高频词。
        *   **去除标点符号和数字**。
    2.  **数据转换**：
        *   **向量化**：使用 **TF-IDF** 将文本转换为向量空间模型。
        *   **处理罕用语**：PPT第55页建议**剪切掉（删除）罕用语**。
        *   *理由*：只出现一次的词（Hapaxes）通常是拼写错误或极其生僻的词，它们会显著增加特征维度（造成稀疏性），且对发现普遍规律（模型泛化）没有帮助，甚至可能导致过拟合。
    3.  **建模与评估**：
        *   应检查模型中学到的**最具区分度特征（Most Informative Features）**。
        *   如果“科技”和“政治”混淆，可能是因为它们共享了一些高频词（如“政策”、“发展”、“未来”）。
        *   需要寻找专属于某一类的词（如科技类的“芯片”、“AI”，政治类的“选举”、“法案”）。如果发现很多无意义的词权重很高，说明停用词过滤没做好，或者需要引入**二元词条（Bigrams）**来捕捉特定短语（如 "White House" vs "White paper"）。

**2. 搜索词义消歧与知识图谱**
**背景：** PPT第6页提到了谷歌搜索 "Chelsea" 的例子（是切尔西足球俱乐部？还是切尔西·克林顿？还是切尔西这个地名？）。
**问题：**
1.  **问题分析：** 在文本挖掘中，同一个词有多种含义的现象叫什么？（多义性/Polysemy）。
2.  **解决方案：** 搜索引擎如何利用**上下文（Context）**来解决这个问题？请结合PPT第14页谷歌地图对 "Springfield" 的提示，说明搜索引擎需要用户提供什么额外信息？
3.  **知识关联：** PPT第9页展示了Wolfram Alpha引擎。它与传统基于关键词匹配的搜索（如早期的Google）有何本质区别？它是如何利用“结构化数据”回答问题的？

*   **详细解答：**
    1.  **现象**：这被称为**多义性（Polysemy）**或**歧义（Ambiguity）**。
    2.  **利用上下文**：
        *   搜索引擎通过分析查询词周边的词（Co-occurrence）来判断。例如输入 "Chelsea scores"，"scores" 提示了体育背景；输入 "Chelsea Clinton"，"Clinton" 提示了人物背景。
        *   PPT第14页 "Springfield" 的例子说明，当上下文不足时，系统需要用户提供**地理位置信息**（如州名：IL, MA, OR）或利用用户的**历史搜索记录/当前位置**来进行个性化消歧。
    3.  **Wolfram Alpha的区别**：
        *   传统搜索是**基于文档检索**（返回包含关键词的网页链接）。
        *   Wolfram Alpha是**计算知识引擎**（Computational Knowledge Engine）。它不返回网页，而是基于内部构建的**结构化知识库**进行逻辑推理和计算，直接返回确切答案（如PPT第10页直接画出爱因斯坦曲线）。它将自然语言问题转化为结构化查询，这需要极高精度的实体识别和逻辑解析能力。

### 第九章《数据可视化》模拟试题

#### 一、 简答题（共5题，每题6分）

1.  **PPT第4页总结了可视化的四个基本特征：易懂性、必然性、片面性和专业性。请重点解释什么是“片面性”（One-sidedness），这对数据分析师提出了什么警示？**
    *   **参考答案：**
        *   **含义**：数据可视化只是数据的一种特定表达形式，**可视化模式不能替代数据本身**。任何一种可视化图表都只能展示数据的一个侧面或一部分特征，而无法全方位无损地还原原始数据的所有细节。
        *   **警示**：分析师不能完全依赖可视化结果做决策，必须结合原始数据进行验证。同时，在制作图表时，要意识到图表可能会产生误导（如截断坐标轴），需要客观地选择展现形式，避免“以偏概全”。

2.  **简述大数据可视化的基本流程（四个步骤），并说明“可视化模式”这一步的核心任务是什么？**
    *   **考点关联：** PPT第6页。
    *   **参考答案：**
        *   **流程**：1. 数据获取 $\rightarrow$ 2. 数据处理 $\rightarrow$ 3. 可视化模式 $\rightarrow$ 4. 可视化应用。
        *   **核心任务**：可视化模式的选取决定了可视化方案的雏形。这一步需要根据数据的特性（是时间序列、网络结构还是地理空间数据）选择一种特殊的展现形式（如标签云、网络图、电子地图等），建立数据属性到视觉元素（位置、颜色、大小）的映射关系。

#### 二、 计算题（共2题，每题10分，包含进阶难度）

**1. 标签云的字号映射计算（线性 vs 对数映射）—— *常见难度***
**背景：** 你正在制作一个标签云。
*   关键词 "Big Data" 出现了 **1000** 次。
*   关键词 "Hadoop" 出现了 **100** 次。
*   关键词 "Excel" 出现了 **10** 次。
屏幕上允许的最小字号为 **10px**，最大字号为 **50px**。

**问题：**
1.  **线性映射：** 如果采用线性映射（$y = kx + b$），请计算 "Hadoop" (100次) 应该显示的字号大小。（保留一位小数）
2.  **对数映射（进阶）：** 在实际应用中，词频差异往往巨大（长尾分布），线性映射会导致低频词太小看不见。通常采用对数映射。公式为 $Size = A \cdot \log_{10}(Frequency) + B$。
    请建立方程组求出 $A$ 和 $B$（使得10次对应10px，1000次对应50px），然后计算 "Hadoop" (100次) 的字号大小。

*   **详细解答：**
    **1. 线性映射**
    *   设频率为 $f$，字号为 $S$。$S = k \cdot f + b$。
    *   两点确定直线：$(10, 10)$ 和 $(1000, 50)$。
    *   $\begin{cases} 10 = 10k + b \\ 50 = 1000k + b \end{cases}$
    *   相减得：$40 = 990k \Rightarrow k = 40/990 \approx 0.0404$。
    *   代入求b：$b = 10 - 0.404 = 9.596$。
    *   计算 "Hadoop" (f=100)：
        $S = 0.0404 \times 100 + 9.596 = 4.04 + 9.596 = \mathbf{13.6 \text{ px}}$。
    *   *评论：线性映射下，100次和10次的大小非常接近，区分度极差，所以通常不用。*

    **2. 对数映射**
    *   公式：$S = A \cdot \log_{10}(f) + B$。
    *   点1 (f=10, S=10): $10 = A \cdot \log_{10}(10) + B \Rightarrow 10 = A + B$ ①
    *   点2 (f=1000, S=50): $50 = A \cdot \log_{10}(1000) + B \Rightarrow 50 = 3A + B$ ②
    *   解方程组（② - ①）：$40 = 2A \Rightarrow \mathbf{A = 20}$。
    *   代入①：$10 = 20 + B \Rightarrow \mathbf{B = -10}$。
    *   计算 "Hadoop" (f=100)：
        $S = 20 \cdot \log_{10}(100) - 10$
        $S = 20 \cdot 2 - 10 = 40 - 10 = \mathbf{30 \text{ px}}$。
    *   *结论：对数映射下，100次(30px)正好位于10次(10px)和1000次(50px)的中间，视觉效果更好。*

**2. 社交网络图指标计算（图论基础）—— *较难难度***
**背景：** PPT第19页NodeXL部分提到了**网络连通度**和**聚类系数**。现有一个小型的社交网络图，包含5个节点（A, B, C, D, E）。
连接关系（边）如下：
*   A-B, A-C, A-D, A-E (A认识所有人)
*   B-C (B和C互为朋友)
*   D-E (D和E互为朋友)
*(注：这是一个无向图，共6条边)*

**问题：**
1.  **网络密度（Graph Density）：** 计算该图的密度。公式：$\text{Density} = \frac{2E}{N(N-1)}$，其中E是实际边数，N是节点数。
2.  **度中心性（Degree Centrality）：** 哪个节点的度中心性最高？数值是多少？（度数 / N-1）。
3.  **局部聚类系数（Local Clustering Coefficient）：** 计算节点 **A** 的聚类系数。
    *   *定义*：节点A的朋友之间实际存在的边数，除以A的朋友之间**可能存在的最大边数**。
    *   *公式*：$C_i = \frac{2e_i}{k_i(k_i-1)}$，其中 $k_i$ 是邻居数量，$e_i$ 是邻居之间实际连线的数量。

*   **详细解答：**
    **1. 网络密度**
    *   节点数 $N=5$，实际边数 $E=6$。
    *   可能的最大边数 = $5 \times 4 / 2 = 10$。
    *   Density = $6 / 10 = \mathbf{0.6}$。

    **2. 度中心性**
    *   节点 A 的度（连接数）= 4 (B,C,D,E)。
    *   A 的度中心性 = $4 / (5-1) = 4/4 = \mathbf{1.0}$ (或100%)。
    *   *结论：A是绝对的中心节点（Star node）。*

    **3. 节点 A 的聚类系数**
    *   A 的邻居（$k_i$）= 4 个 (B, C, D, E)。
    *   A 的邻居之间可能存在的最大连接数 = $4 \times 3 / 2 = 6$ 条。
    *   A 的邻居之间**实际**存在的连接：
        *   B-C (有)
        *   D-E (有)
        *   B-D, B-E, C-D, C-E (均无)
        *   所以实际连接数 $e_i = 2$。
    *   $C_A = \frac{2}{6} \approx \mathbf{0.33}$。
    *   *解释：这说明A的朋友圈并没有紧密抱团，而是分成了两个小圈子(B-C)和(D-E)。*

**题目：在数据科学项目的最后阶段，我们往往需要将复杂的模型结果（如机器学习的分类边界、聚类结果）“翻译”给非技术背景的决策者看。PPT第4页指出“专业性特征是人们从可视化模型中提取专业知识的环节”。**

*   **答题思路与核心要点：**

    **1. 可视化的桥梁作用（8分）**
    *   **认知降维**：人类大脑对数字不敏感，但对图形（颜色、位置、长度）极度敏感。可视化将枯燥的 `csv` 文件转换为图形，降低了认知门槛。
    *   **信任建立**：对于决策者来说，黑盒的机器学习模型（如神经网络）很难信任。通过可视化（如决策树的可视化、特征重要性条形图），可以解释模型是如何工作的（可解释性），从而赢得信任。
    *   **发现盲点**：PPT第5页提到“数据分析”作用，可视化能帮助发现统计数据中被掩盖的异常值或模式（如辛普森悖论）。

    **2. 抽象与聚合的应用（12分）**
    *   **文本数据的抽象**：PPT第8页的 **标签云**。它没有罗列整篇文章，而是通过**聚合**（计算词频）和**抽象**（字号大小）过滤掉无关信息，直接展示核心主题。
    *   **时空数据的抽象**：PPT第13页的 **时空立方体** 和 **流式地图**。如果画出每一辆车的轨迹，地图会变成一团乱麻。通过**边捆绑（Edge Bundling）**技术（PPT第11、12页），将相似路径的线条捆绑在一起，既保留了趋势，又清晰美观。
    *   **多维数据的抽象**：PPT第16页的 **平行坐标**。它将高维空间（无法画出）抽象为平面上的平行线。虽然数据量大时会混乱，但通过**聚簇（Clustering）**和**密度渲染**，可以让决策者一眼看出数据的分布带（Pattern），而不是关注单个数据点。
    *   **总结**：优秀的大数据可视化不是“画出所有数据”，而是“画出数据的特征和规律”。

### 💡 考前必背：计算题公式清单
*(针对你说的需要用科学计算器，这些公式必须烂熟于心)*

1.  **标准化（Z-Score）**：
    $$z = \frac{x - \mu}{\sigma}$$
    （需要计算均值 $\mu$ 和标准差 $\sigma$）

2.  **欧氏距离（Euclidean Distance）**：
    $$d(x,y) = \sqrt{\sum (x_i - y_i)^2}$$

3.  **混淆矩阵指标**：
    *   $Accuracy = \frac{TP + TN}{TP + TN + FP + FN}$
    *   $Precision = \frac{TP}{TP + FP}$ （查准率）
    *   $Recall = \frac{TP}{TP + FN}$ （查全率）
    *   $F1 = \frac{2 \times P \times R}{P + R}$

4.  **均方误差（MSE）**：
    $$MSE = \frac{1}{n} \sum (y_{true} - y_{pred})^2$$

5.  **TF-IDF（文本挖掘）**：
    $$TF = \frac{\text{词在文档中出现的次数}}{\text{文档总词数}}$$
    $$IDF = \log(\frac{\text{文档总数 } N}{\text{包含该词的文档数}})$$
    $$TF\text{-}IDF = TF \times IDF$$

6.  **HDFS存储估算**：
    *   Block数 = $\lceil \text{文件大小} / \text{BlockSize} \rceil$
    *   NameNode对象数 ≈ 文件数 + Block数
    *   NameNode内存 ≈ 对象数 $\times$ 150 Bytes

7.  **分布式系统可用性（CAP相关）**：
    *   $P(\text{系统可用}) = (1 - P(\text{节点故障}))^N$ （无冗余情况）
    *   $P(\text{分片不可用}) = P(\text{节点故障})^{\text{副本数}}$ （有冗余情况）

### 📘 Set A：架构与原理篇 —— 详细答案

#### 一、 简答题（每题6分）

1.  **云计算服务模式**
    *   **答案**：
        1.  **IaaS（基础设施即服务）**：提供底层的计算资源（如虚拟机、存储、网络）。用户需要自己管理操作系统和应用。
        2.  **PaaS（平台即服务）**：提供开发和运行环境（如数据库、中间件）。用户只关注应用程序开发。
        3.  **SaaS（软件即服务）**：提供直接可用的软件应用（如Email、CRM）。
        4.  **Hadoop集群归属**：通常属于 **PaaS**（如果是购买云厂商托管的Hadoop服务，如华为云MRS）或 **IaaS**（如果是自己买云服务器搭建）。在PPT语境中，如果强调“资源池”，通常指IaaS/PaaS层面的支撑。

2.  **异常值检测**
    *   **答案**：
        *   **定义**：异常值是指那些远离其他观测值的数据点，它们遵循不同的逻辑或生成过程（PPT第22页）。
        *   **检测方法**：
            1.  **可视化法**：使用**散点图**（观察偏离回归线的点）或**箱线图**（观察超出上下边缘的点）。
            2.  **统计法**：检查最大值/最小值是否符合业务逻辑（如年龄299岁）。

3.  **HDFS架构与小文件问题**
    *   **答案**：
        *   **职责**：**NameNode**是主节点，负责管理元数据（文件目录树、块位置信息），存储在内存中；**DataNode**是从节点，负责实际存储数据块（Block）。
        *   **小文件问题**：HDFS不适合存储海量小文件，因为每个文件无论大小，在NameNode内存中都占用约 **150字节** 的元数据。海量小文件会迅速耗尽NameNode的内存，而此时DataNode的磁盘空间可能还剩余很多，导致集群扩展性受限（PPT第11页）。

4.  **CAP定理与取舍**
    *   **答案**：
        *   **内容**：一个分布式系统无法同时满足 **一致性（Consistency）**、**可用性（Availability）** 和 **分区容错性（Partition Tolerance）**。
        *   **P的必要性**：分布式系统必然涉及网络通信，网络分区（断网）是不可避免的客观事实，所以必须保证P。
        *   **取舍**：架构师需要在 **CP**（牺牲可用性，保证数据一致，如银行）和 **AP**（牺牲强一致性，保证服务可用，如社交网络）之间做选择。

5.  **Spark vs MapReduce**
    *   **答案**：
        *   **核心优势**：Spark是基于**内存计算**的，而MapReduce是基于磁盘的。
        *   **机制**：Spark引入了 **RDD（弹性分布式数据集）**。它允许将中间计算结果缓存在内存中，避免了MapReduce在Shuffle和迭代过程中频繁的磁盘读写，因此在迭代算法（如机器学习）中速度快得多（PPT第22页）。

#### 二、 计算题（每题10分）

**1. HDFS存储与内存估算**
*   **解题步骤**：
    1.  **NameNode内存计算**：
        *   文件数 = $10^8$ (1亿)。
        *   每个文件对应1个文件对象 + 1个Block对象（因为10KB < 128MB，独占逻辑块ID）= 2个对象。
        *   总对象数 = $10^8 \times 2 = 2 \times 10^8$。
        *   所需内存 = $2 \times 10^8 \times 150 \text{ Bytes} = 300 \times 10^8 \text{ Bytes} \approx 30 \text{ GB}$。
        *   **结论**：**16GB内存不够用**（30GB > 16GB）。
    2.  **物理磁盘占用**：
        *   逻辑大小 = $10^8 \times 10 \text{ KB} = 10^9 \text{ KB} \approx 1 \text{ TB}$。
        *   物理占用（3副本）= $1 \text{ TB} \times 3 = \mathbf{3 \text{ TB}}$。
    3.  **合并大文件后的变化**：
        *   总数据量 1TB 不变。
        *   合并后文件大小 256MB。每个文件占用 2个 Block (256/128=2)。
        *   新文件数量 = $1 \text{ TB} / 256 \text{ MB} = 1024 \times 1024 / 256 = 4096$ 个。
        *   新对象数 = $4096 \times (1 \text{文件} + 2 \text{块}) \approx 1.2 \text{万个}$。
        *   新内存需求 = $1.2 \text{万} \times 150 \text{ B} \approx 1.8 \text{ MB}$。
        *   **结论**：内存压力**急剧下降**（从30GB降至几MB），彻底解决了内存瓶颈。

**2. 机器学习评估指标**
*   **解题步骤**：
    *   TP=15, FP=80.
    *   Total Positive = 20 $\rightarrow$ FN = 20 - 15 = 5.
    *   Total Negative = 980 $\rightarrow$ TN = 980 - 80 = 900.
    1.  **准确率 (Accuracy)**：
        $$Acc = \frac{TP+TN}{Total} = \frac{15+900}{1000} = \mathbf{0.915} \text{ (91.5\%)}$$
    2.  **精确率 & 召回率**：
        $$Precision = \frac{TP}{TP+FP} = \frac{15}{15+80} = \frac{15}{95} \approx \mathbf{0.158} \text{ (15.8\%)}$$
        $$Recall = \frac{TP}{TP+FN} = \frac{15}{20} = \mathbf{0.75} \text{ (75\%)}$$
    3.  **分析**：
        *   **不合格**。虽然准确率高达91.5%，但这完全是因为健康人（负样本）太多了。
        *   精确率极低（15.8%），意味着每抓100个人，只有15个真有病，误诊率太高。
        *   **欺骗性**：在**类别不平衡**（Imbalanced Data）场景下，准确率会掩盖模型无法识别少数类（患病者）的事实。

### 📘 Set B：算法与应用篇 —— 详细答案

#### 一、 简答题（每题6分）

1.  **过拟合**
    *   **答案**：
        *   **定义**：模型在训练集上表现太好，但在测试集上表现差，学到了噪音而非规律。
        *   **防止手段**：1. **正则化**（L1/L2，惩罚复杂模型）；2. **剪枝**（决策树中去掉不必要的分支）；3. **增加训练数据**；4. **交叉验证**。

2.  **在线学习**
    *   **答案**：
        *   **定义**：一种模型训练模式，每次只读取一个（或一批）样本进行参数更新，用完即丢，不保存历史数据。
        *   **解决内存问题**：单机内存有限，无法一次性加载海量数据。在线学习将数据视为“流”，**化整为零**，对内存要求极低，因此适合Out-of-Core场景（PPT第24页）。

3.  **多度人脉查询与图数据库**
    *   **答案**：
        *   **RDBMS瓶颈**：查询“朋友的朋友的朋友”需要进行多次表**连接（Join）**。随着深度增加，Join操作的计算量呈指数级爆炸，导致性能不可接受。
        *   **图数据库解决**：采用**免索引邻接**。每个节点直接持有指向邻居的物理指针。查询时只需顺着指针遍历（Traversal），复杂度只与关联数据的数量有关，而与总数据量无关。

4.  **TF-IDF与停用词**
    *   **答案**：
        *   **TF-IDF含义**：衡量一个词对一篇文档的重要性。TF表示词频（出现越多越重要），IDF表示逆文档频率（在所有文档中越少见越重要）。
        *   **删除停用词**：像 "the", "is" 这样的词TF很高（出现多），但IDF极低（到处都有），几乎没有区分度。删除它们可以**减少特征维度**，**节省存储**，让模型专注更有意义的实词。

5.  **多维可视化**
    *   **答案**：
        1.  **平行坐标图（Parallel Coordinates）**：将多维数据映射到多条平行轴上，适合发现属性间的关系。
        2.  **散点图矩阵（Scatter Plot Matrix）**：展示所有两两变量组合的散点图。
        3.  **Chernoff Faces（切尔诺夫脸谱图）**：用脸部特征代表数据维度（PPT虽未重墨但属常识，PPT重点推了平行坐标）。

#### 二、 计算题（每题10分）

**1. 文本相似度计算**
*   **解题步骤**：
    *   词典：`[Big, Data, is, Good, Fast, Bad]` (共6维)
    1.  **词向量**：
        *   A ("Big Data is Good"): `[1, 1, 1, 1, 0, 0]`
        *   B ("Big Data is Fast"): `[1, 1, 1, 0, 1, 0]`
    2.  **欧氏距离**：
        *   $d = \sqrt{(1-1)^2 + (1-1)^2 + (1-1)^2 + (1-0)^2 + (0-1)^2 + (0-0)^2}$
        *   $d = \sqrt{0+0+0+1+1+0} = \sqrt{2} \approx \mathbf{1.414}$
    3.  **汉明距离**：
        *   不一样的位数有：第4位（1 vs 0）和第5位（0 vs 1）。
        *   距离 = **2**。
    4.  **Jaccard相似度**：
        *   交集（都有的词）：{Big, Data, is} $\rightarrow$ 3个。
        *   并集（所有出现过的词）：{Big, Data, is, Good, Fast} $\rightarrow$ 5个。
        *   $J = 3/5 = \mathbf{0.6}$。

**2. 朴素贝叶斯计算**
*   **解题步骤**：
    *   目标：求 $P(Spam|Lottery)$。
    *   已知：
        *   $P(Spam) = 0.4$
        *   $P(Normal) = 0.6$
        *   $P(Lottery|Spam) = 0.8$
        *   $P(Lottery|Normal) = 0.1$
    *   **贝叶斯公式展开**：
        $$P(Spam|Lottery) = \frac{P(Lottery|Spam) \cdot P(Spam)}{P(Lottery)}$$
    *   **计算分母 P(Lottery)**（全概率公式）：
        $$P(Lottery) = P(Lottery|Spam)P(Spam) + P(Lottery|Normal)P(Normal)$$
        $$= 0.8 \times 0.4 + 0.1 \times 0.6 = 0.32 + 0.06 = 0.38$$
    *   **计算分子**：
        $$0.8 \times 0.4 = 0.32$$
    *   **最终结果**：
        $$P(Spam|Lottery) = 0.32 / 0.38 = \frac{16}{19} \approx \mathbf{0.842} \text{ (84.2\%)}$$

#### 三、 综合分析题（每题15分）

**1. 推荐系统设计**
*   **答案要点**：
    1.  **图建模**：
        *   节点：`:User`, `:Movie`, `:Director`。
        *   关系：`(:User)-[:LIKES]->(:Movie)`, `(:Movie)-[:DIRECTED_BY]->(:Director)`。
        *   **协同过滤逻辑**：查找“共同邻居”。如果 User A 和 User B 都 `[:LIKES]` 电影 M1 和 M2，说明他们相似。那么 User B 喜欢的 M3，很可能 User A 也喜欢。
        *   Cypher思路：`MATCH (u1)-[:LIKES]->(m)<-[:LIKES]-(u2)-[:LIKES]->(rec_m) ...`
    2.  **LSH优化**：
        *   **问题**：1亿用户两两比较复杂度是 $O(N^2)$，无法计算。
        *   **LSH方案**：使用 **MinHash + LSH**。将用户的观影历史（稀疏向量）通过哈希函数映射到不同的“桶”中。
        *   **原理**：只有相似的用户才会大概率掉进同一个桶。我们只需要在**桶内**进行局部搜索和比较，极大减少了计算量，实现了亚线性时间复杂度的查找。

**2. 文本分类实战**
*   **答案要点**：
    1.  **特征工程**：
        *   预处理：分词、统一小写、去停用词。
        *   **N-gram作用**：保留词序信息。例如 "White House"（白宫）作为二元词条（Bigram）比单独的 "White" 和 "House" 更能代表“政治”主题。
    2.  **模型选择**：
        *   选择 **朴素贝叶斯**。
        *   *理由*：尽管它假设特征独立（这在文本中通常不成立），但在实际应用中，它对高维稀疏数据（文本词袋）表现极好，计算速度快，需要的训练数据相对较少，且不易过拟合（PPT第26页）。
    3.  **结果分析**：
        *   检查 **Feature Weights（特征权重）**。可能发现 "Final" 这个词在政治类数据中（如“Final Decision”）也频繁出现，导致权重混淆。
        *   **解决方案**：引入更多实体识别（识别 "NBA" 为体育实体）或增加训练数据。

#### 四、 论述题（20分）

**答案思路（协作流程）**：

这是一个典型的**大数据Lambda架构**或数据处理流水线：

1.  **数据采集与存储（NoSQL & HDFS）**：
    *   用户的点击流和日志是海量且实时的非结构化数据。我们使用 **NoSQL（如HBase/Kafka）** 进行高速写入（利用其高吞吐、低延迟特性）。
    *   历史全量数据会被归档到 **HDFS** 中，作为长期存储（低成本、高可靠）。

2.  **数据处理与计算（MapReduce/Spark）**：
    *   **离线训练**：利用 **MapReduce** 或 **Spark** 从HDFS读取历史数据，进行大规模的数据清洗、ETL和特征提取（如计算用户画像）。Spark的内存计算能加速这一迭代过程。
    *   这些处理后的数据被用来训练 **机器学习模型**。

3.  **智能分析（机器学习）**：
    *   **机器学习算法**（如协同过滤、逻辑回归）在Spark MLlib上运行，利用清洗后的数据训练推荐模型。
    *   模型学习用户特征和商品特征之间的**相关性**。

4.  **服务应用（NoSQL & 可视化）**：
    *   训练好的模型参数或推荐结果列表会被写回 **NoSQL（如Redis）**，以便前端应用能毫秒级读取推荐结果（KV存储优势）。
    *   最后，管理层通过 **可视化仪表盘**（读取聚合后的数据）监控系统效果（如转化率），完成闭环。

**总结**：NoSQL负责“快进快出”，MapReduce/Spark负责“大吞大吐”的深加工，机器学习负责“提炼智慧”。三者各司其职，共同支撑大数据应用。
