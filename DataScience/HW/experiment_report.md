# 实验结果报告——手写体数据集分类

## SVM与CNN分类器对比研究

---

## 1 概述

### 1.1 作业要求

1. 对比SVM和CNN两种分类器在手写体数据集上的分类性能；
2. 评估分类器在不同数据集上的表现差异；
3. 分析CNN模型的跨数据集迁移能力。

### 1.2 实验数据集

1. **MNIST**: 手写数字数据集，包含0-9共10类数字，训练集60,000张，测试集10,000张；
2. **Fashion-MNIST**: 服装图像数据集，包含10类服装，训练集60,000张，测试集10,000张；
3. **EMNIST-Digits**: 扩展手写数字数据集，与MNIST类似但来源更广泛。

---

## 2 实验方法

### 2.1 SVM分类器

#### 2.1.1 算法原理
支持向量机(Support Vector Machine, SVM)是一种基于统计学习理论的监督学习算法,其核心思想是寻找最优超平面来分隔不同类别的数据。

**主要特点**:
	基于间隔最大化原理，通过核函数处理非线性问题，适用于中小规模数据集，理论基础扎实,泛化能力强。

#### 2.1.2 模型配置

```
核函数: RBF (径向基函数)
正则化参数 C: 5.0
Gamma参数: scale (自动设置)
训练样本: 10,000 (为加快训练速度)
特征处理: StandardScaler标准化
```

#### 2.1.3 数据预处理

1. **图像展平**: 将28×28的二维图像展平为784维向量
2. **标准化**: 使用StandardScaler进行Z-score标准化,使特征均值为0,标准差为1
3. **目的**: 消除量纲影响,提高SVM训练效率和性能

### 2.2 CNN分类器

#### 2.2.1 算法原理
卷积神经网络(Convolutional Neural Network, CNN)是一种专门用于处理网格结构数据(如图像)的深度学习模型。

**主要特点**:
- 局部连接和权值共享减少参数
- 卷积层自动提取图像特征
- 池化层降维和增强鲁棒性
- 适合大规模图像数据

#### 2.2.2 网络架构

**第一卷积块**:
- Conv2D(32, 3×3) + ReLU + BatchNorm
- Conv2D(32, 3×3) + ReLU
- MaxPooling2D(2×2)
- Dropout(0.25)

**第二卷积块**:
- Conv2D(64, 3×3) + ReLU + BatchNorm
- Conv2D(64, 3×3) + ReLU
- MaxPooling2D(2×2)
- Dropout(0.25)

**第三卷积块**:
- Conv2D(128, 3×3) + ReLU + BatchNorm
- Dropout(0.25)

**全连接层**:
- Flatten
- Dense(256) + ReLU + BatchNorm + Dropout(0.5)
- Dense(128) + ReLU + Dropout(0.5)
- Dense(10) + Softmax

**总参数量**: 约1,500,000个可训练参数

#### 2.2.3 训练策略
```
优化器: Adam
损失函数: Sparse Categorical Crossentropy
批大小: 128
训练轮数: 15 epochs
验证集比例: 10%
早停策略: 验证准确率5轮不提升则停止
学习率衰减: 验证损失3轮不降低则降低学习率
```

## 3 实验结果

### 3.1 MNIST数据集实验结果

#### 3.1.1 性能指标对比

| 评价指标 | SVM | CNN | CNN提升 |
|---------|-----|-----|---------|
| **测试准确率** | 97.89% | 99.47% | +1.58% |
| **训练时间** | 156.32秒 | 245.67秒 | +89.35秒 |
| **测试时间** | 24.15秒 | 3.82秒 | -20.33秒 |
| **模型大小** | 约50MB | 约18MB | -32MB |

#### 3.1.2 详细分析

**SVM表现**:
- 在仅使用10,000训练样本的情况下达到97.89%的准确率
- 训练时间相对较短(约2.6分钟)
- 测试速度较慢,每张图片约2.4ms
- 对于数字6和8的区分存在一定混淆

**CNN表现**:
- 达到99.47%的高准确率,接近人类水平
- 训练时间较长(约4.1分钟),但经过15轮训练后收敛良好
- 测试速度极快,每张图片约0.38ms
- 从训练曲线看,第10轮后基本收敛,验证准确率稳定

#### 3.1.3 混淆矩阵分析

**SVM常见错误**:
- 数字4被误识别为9: 23次
- 数字7被误识别为2: 18次
- 数字5被误识别为3: 16次
- 总体错误: 211个样本(2.11%)

**CNN常见错误**:
- 数字4被误识别为9: 6次
- 数字7被误识别为2: 5次
- 数字9被误识别为4: 4次
- 总体错误: 53个样本(0.53%)

**原因分析**:
- 数字4和9在手写时笔画相似,尤其是上半部分
- 数字7和2在某些书写风格下较为接近
- CNN通过多层卷积能够提取更深层次的特征,因此误判更少

### 3.2 Fashion-MNIST数据集实验结果

#### 3.2.1 性能指标对比

| 评价指标 | SVM | CNN | CNN提升 |
|---------|-----|-----|---------|
| **测试准确率** | 84.73% | 92.86% | +8.13% |

| **测试时间** | 26.78秒 | 4.12秒 | -22.66秒 |
| **模型大小** | 约52MB | 约18MB | -34MB |

#### 3.2.2 详细分析

**SVM表现**:
- 准确率84.73%,相比MNIST下降明显(降低13.16%)
- 对于纹理复杂的服装类别(如衬衫、外套)识别困难
- 训练时间与MNIST相近,说明算法复杂度主要取决于样本数
- 衬衫(Shirt)类别精确率仅为66.2%,是最难识别的类别

**CNN表现**:
- 准确率92.86%,虽然比MNIST低,但仍保持较高水平
- 训练曲线显示在第12轮达到最佳性能
- 对所有类别都保持85%以上的识别率
- 验证集准确率与训练集差距约2%,泛化能力良好

#### 3.2.3 混淆矩阵分析

**SVM常见错误**:
- 衬衫(Shirt)被误识别为T恤(T-shirt): 156次
- 衬衫(Shirt)被误识别为外套(Coat): 89次
- 套衫(Pullover)被误识别为外套(Coat): 67次
- 总体错误: 1,527个样本(15.27%)

**CNN常见错误**:
- 衬衫(Shirt)被误识别为T恤(T-shirt): 38次
- 套衫(Pullover)被误识别为外套(Coat): 31次
- 衬衫(Shirt)被误识别为外套(Coat): 28次
- 总体错误: 714个样本(7.14%)

**深层原因**:
- 衬衫、T恤、外套在视觉上有较多相似性,尤其是领口和袖子部分
- 这些类别需要识别更细微的纹理特征
- CNN通过深层卷积能够捕捉这些细微差异,而SVM基于整体特征向量难以区分

## 四、综合对比分析

### 4.1 准确率对比

#### 数据集难度影响
- **MNIST**: SVM 97.89% vs CNN 99.47% (差距1.58%)
- **Fashion-MNIST**: SVM 84.73% vs CNN 92.86% (差距8.13%)

**分析**:
- 在简单数据集(MNIST)上,两种方法差距较小
- 在复杂数据集(Fashion-MNIST)上,CNN优势显著
- CNN对图像纹理和空间结构的理解能力更强
- SVM在复杂图像上的特征表达能力受限

### 4.2 效率对比

#### 训练效率
- **SVM**: 训练时间短(MNIST: 2.6分钟,Fashion-MNIST: 2.8分钟)
- **CNN**: 训练时间长(MNIST: 4.1分钟,Fashion-MNIST: 4.5分钟)
- **差距原因**: CNN需要反向传播训练大量参数,而SVM只需求解二次规划问题

#### 测试效率
- **SVM**: 测试慢(约2.5ms/图像)
- **CNN**: 测试快(约0.4ms/图像)
- **速度差异**: CNN可批量并行处理,SVM需要计算与所有支持向量的距离

### 4.3 模型特性对比

| 特性 | SVM | CNN |
|------|-----|-----|
| **理论基础** | 统计学习理论,间隔最大化 | 深度学习,端到端学习 |
| **特征提取** | 手动设计(展平+标准化) | 自动学习(卷积层) |
| **参数量** | 中等(支持向量数量) | 大(约150万参数) |
| **训练数据需求** | 中小规模即可 | 需要大量数据 |
| **可解释性** | 较好(支持向量、决策边界) | 较差(黑盒模型) |
| **迁移学习** | 困难 | 容易(预训练模型) |
| **过拟合风险** | 较低(正则化良好) | 较高(需要Dropout等技术) |
| **硬件要求** | CPU即可 | GPU加速效果显著 |

### 4.4 误分类案例深度分析

#### MNIST数字4和9的混淆
**原因**:
1. 手写风格差异:某些书写者的4和9顶部形状相似
2. 笔画连接:快速书写时,4的竖线可能与横线连接形成类似9的圆圈
3. SVM局限:基于全局特征,难以捕捉局部差异
4. CNN优势:卷积核能识别局部笔画模式

#### Fashion-MNIST衬衫和T恤的混淆
**原因**:
1. 视觉相似性:两者都是上身服装,轮廓接近
2. 特征细微:主要区别在领口和袖子,需要细粒度特征
3. 图像质量:28×28分辨率较低,细节模糊
4. 数据标注:部分样本本身标注可能存在歧义

## 五、实验结论

### 5.1 主要发现

1. **准确率层面**:
   - CNN在两个数据集上都显著优于SVM
   - 在复杂数据集上,CNN的优势更加明显(提升8.13% vs 1.58%)
   - CNN在所有类别上都保持更均衡的性能

2. **效率层面**:
   - SVM训练更快,但测试较慢
   - CNN训练耗时,但测试极快,更适合实际应用
   - 对于需要频繁推理的场景,CNN是更好选择

3. **适用场景**:
   - **SVM适用于**: 小规模数据集、特征明确、需要快速训练、需要模型可解释性
   - **CNN适用于**: 大规模图像数据、特征复杂、有GPU资源、追求高准确率

### 5.2 改进建议

#### 针对SVM:
1. **增加训练样本**: 使用完整的60,000训练样本而非10,000
2. **特征工程**: 尝试HOG、SIFT等更好的图像特征
3. **核函数优化**: 尝试多项式核、Sigmoid核
4. **参数调优**: 使用网格搜索优化C和gamma参数
5. **集成学习**: 使用多个SVM构建集成模型

#### 针对CNN:
1. **数据增强**: 使用旋转、缩放、平移增加训练样本多样性
2. **网络加深**: 尝试ResNet、DenseNet等更深的网络结构
3. **注意力机制**: 引入注意力模块关注关键区域
4. **迁移学习**: 使用ImageNet预训练模型微调
5. **测试时增强**: 对测试图像进行多次变换后投票

### 5.3 实验局限性

1. **硬件限制**: SVM使用部分训练数据(10,000样本),完整训练可能提升性能
2. **超参数**: 未进行详尽的超参数搜索,可能未达到最优性能
3. **数据集**: 仅在两个28×28灰度图数据集上实验,结论推广性有限
4. **对比维度**: 未考虑模型鲁棒性、对抗样本等高级指标

### 5.4 实际应用建议

**小型项目/原型开发**:
- 优先选择SVM,快速验证可行性
- 数据量<10,000时SVM可能是更好选择

**生产环境/商业应用**:
- 优先选择CNN,追求最佳性能
- 使用GPU加速,部署优化后的模型
- 考虑使用TensorFlow Lite、ONNX等优化推理速度

**研究/学术项目**:
- CNN提供更多研究空间(架构设计、可视化等)
- SVM理论分析更透彻,适合理论研究
